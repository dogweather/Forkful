## 何となぜ？

プログラミングで乱数を生成することは、シミュレーションの作成、テスト、暗号化、ゲームなどにおいて重要です。Gleamでは、開発者がアプリケーションに予測不可能性を導入したり、現実世界のシナリオをシミュレートしたりする機能を提供します。

## 方法：

Gleamで乱数を生成するには、主に`gleam_random`ライブラリを使用します。このライブラリは乱数の整数、浮動小数点数などを生成する関数を提供します。まず、依存関係として`gleam_random`を`rebar.config`や`mix.exs`ファイルに追加したことを確認してください。

いくつかの例に触れましょう：

### 乱数の整数を生成する

指定された範囲内で乱数の整数を生成するには、`int`関数を使用します：

```gleam
import gleam/random

pub fn generate_random_int() {
  let random_int = random.int(1, 10)
  random_int
}
```

この関数は1から10の間の乱数の整数を生成します。

### 乱数の浮動小数点数を生成する

乱数の浮動小数点数を得るには、`float`関数を使用します。これは0.0から1.0の間の浮動小数点数を生成します：

```gleam
import gleam/random

pub fn generate_random_float() {
  let random_float = random.float()
  random_float
}
```

### 例の出力

これらの関数を実行すると、次のような出力が得られるかもしれません：

- `generate_random_int()`の場合：`5`
- `generate_random_float()`の場合：`0.84372`

乱数の性質上、それぞれの実行で異なる出力が得られることを覚えておいてください。

## 詳細解説

`gleam_random`モジュールは擬似乱数生成器（PRNG）を実装しており、本質的には数が真にランダムではないが予測しにくく、ランダム性を模倣しています。PRNGは、シードとして知られる初期値から始まり、数列を生成するために数理演算を適用することで動作します。

歴史的に、言語やライブラリはメルセンヌ・ツイスターや線形合同生成器（LCG）など、PRNGのためにいくつかのアルゴリズムを実装してきました。「乱数」の品質にはアルゴリズムの選択が影響し、中には他のものよりも暗号化用途に適しているものがあります。Gleamの標準ライブラリは`gleam_random`モジュールで便利さと使いやすさを提供しますが、暗号学的に安全な乱数が必要なユースケースでは常に最良の選択とは限りません。暗号目的の場合、開発者は生成された数列を観察することによって今後の数を予測する可能性のある攻撃に耐えるよう設計された、暗号学的に安全な擬似乱数生成器（CSPRNG）を提供するライブラリを探すべきです。

結論として、Gleamの乱数生成機能は一般的なプログラミングニーズには堅牢ですが、特定のセキュリティ要件を持つアプリケーションは、乱数生成の完全性とセキュリティを保証するために専用の暗号解決策を検討する必要があります。
