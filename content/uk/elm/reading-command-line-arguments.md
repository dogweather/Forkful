## Що і чому?

Читання аргументів командного рядка - це процес отримання даних, переданих програмі в момент її запуску. Це необхідно для збільшення гнучкості програми: ми можемо міняти її поведінку без зміни коду.

## Як використовувати:

Elm не має прямого доступу до аргументів командного рядка, але можливий підхід - обробляти їх з JavaScript і передавати в Elm через флаги.

```javascript
**Elm.Main.init({ flags: process.argv });**
```

Переданий флаг потрібно в Elm моделі згадати.

```elm
type alias Model = 
    { cmdArgs : List String }
```

Тут, cmdArgs - це список зрядків отриманих з командного рядка.

## Детальніше:

1. **Історичний контекст**: Початкові версії Elm не розглядали концепції аргументів командного рядка. Але з часом Elm стали застосовувати інтенсивніше в серверних застосунках, тому було реалізовано використання командного рядка через флаги і JavaScript.
   
2. **Альтернативи**: Є можливість обійтися без передачі аргументів командного рядка, використовуючи конфігураційні файли або взаємодію через API.
   
3. **Деталі реалізації**: Elm використовує аргументи командного рядка, передані йому через флаги з JavaScript. Швидше за все, це й контекст безпосереднього доступу до командного рядка не має сам Elm.
