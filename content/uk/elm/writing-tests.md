## Чому
Люди часто питають, чому нам потрібні тести? На перший погляд, це може здатися зайвим, але насправді тести є надзвичайно важливим інструментом для підтримки якості коду та попередження можливих помилок.

## Як це зробити
Є кілька способів, які допоможуть вам зрозуміти і вивести у віповідне матеріали в якийсь з пулінгу. Давайте подивимося на кілька прикладів коду Elm та висзначимо вивід в console.log у "``` Elm...```" блоках.

```Elm
import Html exposing (h1, div, text)
import Console

main =
   h1 []
      [ text "Hello, World!" ]

Console.log "Це повідомлення буде виведене в консоль!"
```
В цьому прикладі ми імпортуємо Html бібліотеку та створюємо простий текст "Hello, World!" за допомогою функції `h1`, який буде виведений на екран. Ми також використовуємо функцію `Console.log`, щоб вивести повідомлення у консоль. Це допоможе вам відстежувати дані та знаходити можливі помилки в вашому коді.

```Elm
import Html exposing (h1, div, text)
import Html.Attributes exposing (style)

main =
   div [ style [("text-align", "center")] ]
      [ h1 [] [ text "Це текст буде центрованим!" ]
      , div [ style [("color", "blue")] ] [ text "Це текст буде синім!" ]
      ]

Console.log "Це повідомлення буде виведене в консоль!"
```

Цей приклад демонструє використання атрибутів у функції `div` та `h1`. Ми використовуємо атрибут `style` для визначення того, як буде відображатися текст. Ви можете використовувати багато різних атрибутів для стилізації своїх елементів.

## Глибока проникнення
Написання тестів важливо для забезпечення якості коду та швидкої валідації. Це допомагає уникнути проблем у великому масштабі пізніше в процесі розробки. Тести також допомагають розробникам зберігати свій код документованим та зміщеним.
