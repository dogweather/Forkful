## Що та чому?

Виведення налагоджувальної інформації - це техніка, якою програмісти друкують значення змінних або повідомлення в процесі виконання програми. Такий підхід допомагає виявити та виправити помилки в коді.

## Як зробити:

```Python
# Найпростіший приклад цього - використання функції print()
x = 10
print(f'Значення x: {x}')
```

Виходить:

```
Значення x: 10
```

Ми можемо також використовувати модуль logging для більшої гнучкості:

```Python
import logging

logging.basicConfig(level=logging.DEBUG)
logging.debug(f'Значення x: {x}')
```

Перевага використання logging полягає в тому, що ви можете легко включати чи відключати відладку, змінюючи рівень логування.

## Пірнання углиб:

Історично виведення налагоджувальної інформації допомагало розробникам відслідковувати програму "в дії", особливо коли бракувало сучасних відлагоджувачів.

В альтернативах до `print()` або `logging.debug()`, можна розглянути застосування відладки Python (pdb), який пропонує гораздо більше можливостей, але може бути більш складним у використанні.

Наведені приклади хоча і прості, але вони використовують вбудовані функції Python для виведення тексту в стандартний вивід або дозволяють записувати інформацію в файл логів для подальшого вивчення.
