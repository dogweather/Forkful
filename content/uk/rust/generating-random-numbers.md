## What & Why? / Що і чому?

Генерація випадкових чисел — це спосіб створення чисел, які не можна передбачити. Програмісти використовують їх для тестування, ігор, криптографії та імітації відповіді на вхідні дані.

## How to: / Як робити:

```Rust
use rand::Rng; // Додаємо бібліотеку rand

fn main() {
    let mut rng = rand::thread_rng(); // Створюємо генератор для поточного потока
    let random_number: u32 = rng.gen(); // Генеруємо випадкове ціле число
    println!("Random u32: {}", random_number); // Виводимо число

    let random_float: f64 = rng.gen(); // Генеруємо випадкове число з плаваючою комою
    println!("Random f64: {}", random_float);

    let die_roll: u8 = rng.gen_range(1..=6); // Генеруємо число від 1 до 6
    println!("Roll the die: {}", die_roll);
}
```
Вивід прикладу (кожен запуск унікальний):
```
Random u32: 217475123
Random f64: 0.6567894388656623
Roll the die: 4
```

## Deep Dive / Поглиблений розбір:

Історично, випадкові числа генерувалися апаратно — природним шумом чи рухом геїгерівських лічильників. У компютерах вони називаються псевдовипадковими, бо вираховуються алгоритмами. На відміну від справжніх випадкових чисел, псевдовипадкові числа можуть повторюватися, якщо розпочати з однакового "насіння" (seed).

В Rust, бібліотека `rand` — стандарт де-факто для генерації випадкових чисел. Ця бібліотека надає зручні абстракції для роботи з випадковістю. Але є й інші методи, наприклад, оперування системними джерелами випадковості або застосування криптографічно безпечних випадкових чисел з бібліотеки `rand::rngs::OsRng`.

Імплементація в `rand` може включати декілька генераторів випадкових чисел. Наприклад, `thread_rng()` для загальних цілей, який використовується в нашому прикладі, або `rand::rngs::StdRng`, який може бути конфігурований з конкретним насінням для відтворюваної випадковості.
