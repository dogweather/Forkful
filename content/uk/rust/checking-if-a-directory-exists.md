## Що і чому?
Перевірка наявності каталогу в Rust — це спосіб дізнатися, чи реально існує певна директорія на файловій системі. Програмісти роблять це, щоб уникнути помилок при спробах доступу чи зміні файлів у неіснуючих папках.

## Як це зробити:
```Rust
use std::path::Path;

fn main() {
    let path = Path::new("/шлях/до/вашого/каталогу");

    if path.exists() {
        println!("Каталог існує!");
    } else {
        println!("Каталог не знайдено.");
    }
}
```

Вищенаведений код спробує знайти каталог за вказаним шляхом і виведе повідомлення залежно від результату.

## Поглиблений Розбір:
Техніка перевірки існування каталогу виникла разом з потребою управління файловими системами. В Rust це особливо важливо через його гарантії безпеки пам’яті і запобігання гонитві за ресурсами. Альтернативи `path.exists()` включають методи як `path.is_dir()`, який перевіряє, чи шлях дійсно вказує на каталог (а не на файл), та роботу з поверненням помилок через `fs::metadata()` для отримання більш детальної інформації. Ці методи дають більше контролю над тим, як ми хочемо взаємодіяти з файловою системою.
