## Чому

Написання тестів є ключовою складовою у процесі розробки програмного забезпечення. Вони дозволяють перевірити правильність роботи коду та запобігти появі помилок у ранніх етапах розробки. Це ефективний спосіб збільшити якість вашого програмного продукту та зекономити час у майбутньому.

## Як

Щоб написати тести у Rust, вам спочатку потрібно встановити мову програмування та інструменти для написання тестів. Наступним кроком буде створення функції, яку ви будете тестувати, та написання тестових блоків, що перевірятимуть правильність її роботи. Ось приклад коду та вихідного повідомлення у Rust:

```Rust
fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
        assert_eq!(add(10, -5), 5);
        assert_eq!(add(-8, -3), -11);
    }
}
```

Ви можете бачити, що кожен тест містить опис очікуваного результату за допомогою функції `assert_eq`. Якщо в результаті будь-яких змін у вашому коді тести не проходять, це свідчить про наявність помилок та зобов'язує вас розібратися у них.

## Поглиблене вивчення

Щоб написати якісні тести, важливо розуміти принципи їх створення. Варто використовувати максимальну покриття коду тестами, щоб перевірити всі можливі шляхи виконання програми. Також, бажано використовувати правильні дані для тестування, щоб впевнитися у коректному виконанні програми на будь-яких вхідних даних.
