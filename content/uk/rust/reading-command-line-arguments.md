# Читання аргументів командного рядка в Rust

## Що і Навіщо?

Читання аргументів командного рядка – це процес отримання даних, що вводяться користувачем при виклику програми. Програмісти роблять це, щоб управляти типами операцій, які буде виконувати програма на основі цих вхідних даних.

## Як це зробити:

Ось вам простий приклад коду, який читає аргументи командного рядка:

```Rust
fn main() {
    let args: Vec<String> = std::env::args().collect();
    println!("{:?}", args);
}
```

Якщо ви запустите цю програму з командного рядка, побачите такий вивід:

```Shell
$ cargo run arg1 arg2
["target/debug/my_program", "arg1", "arg2"]
```

## Пірнаємо глибше

Читання аргументів командного рядка то старий метод, який використовувався ще з часів Unix. Але маємо декілька альтернатив в Rust. Окрім методу `std::env::args()`, ми можемо використовувати `std::env::args_os()` для ситуацій, коли нам потрібно розглядати аргументи, які містять дані, що не є Unicode.

Використовуючи `std::env::args()`, аргументи програми читаються в `Vec<String>`. Другими словами, колекція `String` об'єктів. Важливо зазначити, що перший елемент у цьому векторі - це шлях до виконуваного файлу.
