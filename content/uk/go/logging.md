## Що та чому?
Логування - це процес ведення записів подій, станів та потоків даних у програмі. Програмісти це роблять для діагностики помилок, моніторингу продуктивності та відстеження оперативного стану додатку — практично роблячи його програмним еквівалентом чорної скриньки в літаках.

## Як робити:
У Go логування можна обробляти кількома способами, починаючи від стандартної бібліотеки `log` до бібліотек від сторонніх розробників, таких як `logrus` та `zap`. Ось простий приклад використання вбудованого пакету `log`:

```Go
package main

import (
	"log"
	"os"
)

func main() {
	// Створення файлу логу
	logFile, err := os.OpenFile("app.log", os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0666)
	if err != nil {
		log.Fatal(err)
	}
	defer logFile.Close()

	// Встановлення виводу логів у файл
	log.SetOutput(logFile)

	// Запис подій у лог
	log.Println("Запуск програми...")
	// ... логіка програми тут ...
	log.Println("Програма успішно завершена.")
}
```

Якщо ви запустите цей код, ви не побачите виводу в термінал, адже все піде у файл `app.log`. Ось як це може виглядати у файлі логу:

```
2023/01/02 15:04:05 Запуск програми...
2023/01/02 15:05:01 Програма успішно завершена.
```

## Поглиблене вивчення
Логування у програмуванні сягає корінням до найраніших комп'ютерів, де інженери буквально знаходили комах (так, метеликів), що були розчавлені в апаратурі, і фіксували їх! Прискорюючи час до сьогодення, логування стало витонченим способом розуміння того, що відбувається у складних системах.

Хоча `log` пакет у Go є досить примітивним, він може бути достатнім для простих застосувань. Проте, в контексті сучасних розподілених систем, або коли вам потрібен більш тонкий контроль над вашим лог-виводом (як-от різні рівні серйозності), можливо ви захочете вивчити більш розгалужені рішення.

Бібліотеки від сторонніх розробників, такі як `logrus` та `zap`, пропонують структуроване логування, що означає, що ви можете записувати складні типи даних, як то JSON, що робить легшим інтерпретацію логів, особливо в поєднанні з системами управління логами на кшталт ELK Stack або Splunk.

При розгляді впровадження стратегії логування також важливо думати про вплив на продуктивність. Бібліотеки високопродуктивного логування оптимізовані для зменшення впливу на пропускну спроможність та затримки в програмі. Наприклад, `zap` хвалиться своїм швидкісним, низькоалокаційним дизайном, що може бути критичним для систем реального часу.

Крім різноманітних бібліотек, також варті уваги формати та стандарти логування. Структуровані формати логування, як-от JSON, можуть бути надзвичайно потужними, коли використовуються разом із системами обробки логів. З іншого боку, звичайні текстові логи є зрозумілими для людини, але важче їх парсити програмно.
