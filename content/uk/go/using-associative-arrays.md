## Що та чому?

Асоціативні масиви, відомі як мапи в Go, дозволяють вам зберігати та отримувати доступ до даних за допомогою пар ключ-значення. Вони є незамінними для управління колекціями, де ви можете швидко здійснювати пошук значень за унікальним ключем, спрощуючи маніпуляцію з даними та їх отримання у ваших програмах.

## Як користуватися:

У Go мапи прості у використанні. Ось простий посібник для початку роботи:

1. **Оголошення та Ініціалізація Мап**

```Go
package main

import "fmt"

func main() {
    // Ініціалізація порожньої мапи з рядковими ключами та цілочисельними значеннями
    var scores map[string]int
    fmt.Println(scores) // Друкує: map[]

    // Оголошення та ініціалізація непорожньої мапи
    colors := map[string]string{
        "red": "#ff0000",
        "green": "#00ff00",
    }
    fmt.Println(colors) // Друкує: map[green:#00ff00 red:#ff0000]
}
```

2. **Додавання та Доступ до Елементів**

```Go
func main() {
    fruits := make(map[string]int)
    fruits["apples"] = 5
    fruits["bananas"] = 10

    fmt.Println(fruits["apples"]) // Друкує: 5
}
```

3. **Ітерація по Мапах**

```Go
func main() {
    pets := map[string]string{"dog": "bark", "cat": "meow"}

    for key, value := range pets {
        fmt.Printf("%s goes %s\n", key, value)
    }
    // Порядок виводу може варіюватися, оскільки мапи не гарантують порядок.
}
```

4. **Видалення Елементів**

```Go
func main() {
    meals := map[string]int{"breakfast": 300, "lunch": 600}
    fmt.Println(meals) // Перед видаленням

    delete(meals, "lunch")
    fmt.Println(meals) // Після видалення
}
```

## Поглиблений Огляд

Введені у Go 1 мапи забезпечують вбудований спосіб ефективного використання асоціативних масивів. На відміну від слайсів, які є впорядкованими колекціями, мапи є невпорядкованими. Це означає, що порядок ітерації по елементах мапи не гарантується однаковим при кожному виконанні, компроміс за її здатність динамічно обробляти пари ключ-значення з значною гнучкістю.

Під капотом Go реалізує мапи як хеш-таблиці, забезпечуючи середню складність дій доступу, вставки та видалення на рівні O(1) в більшості випадків. Однак варто зазначити, що ця ефективність може варіюватися в залежності від таких факторів, як зіткнення хешів.

Для випадків використання, які вимагають упорядкованої ітерації ключів, ви можете розглянути поєднання мап зі слайсами або дослідження сторонніх пакетів, які пропонують додаткові структури даних, такі як упорядковані мапи або дерева. Незважаючи на їхні обмеження, мапи Go є потужним і незамінним інструментом для багатьох програмувальних сценаріїв.
