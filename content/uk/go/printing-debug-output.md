## Що це таке та навіщо?
Друкування налагоджувального виводу – це як робити нотатки на полях книги, але для коду. Програмісти використовують цю техніку, щоб слідкувати за тим, що відбувається всередині програми під час її роботи.

## Як це зробити:
```go
package main

import (
	"fmt"
	"log"
)

func main() {
	// Простий принт
	fmt.Println("Це працює!")

	// Використання log для більш інформативного виводу
	log.Println("Це повідомлення з журналу логів!")

	// Налаштовуємо log для виведення номера рядка
	log.SetFlags(log.LstdFlags | log.Lshortfile)
	log.Println("Це повідомлення покаже номер рядка та файл!")
}

```
Виведення програми:
```
Це працює!
2023/01/02 15:04:05 Це повідомлення з журналу логів!
2023/01/02 15:04:05 main.go:14: Це повідомлення покаже номер рядка та файл!
```

## Детальніше:
Ще в часи C, позначка `printf` була засобом для налагоджування. Сьогодні, на вибір є купа інструментів: `fmt` для простого друкування і `log` для детального журналювання з додаванням часу, файлів та рядків. `log` також дозволяє налаштувати власний журнал та виведення через `log.SetOutput()`. Щодо налагоджування, важливо пам'ятати про чистоту коду: видаляти налагоджувальний вивід перед продакшн - гарна практика.
