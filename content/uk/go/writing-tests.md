## Чому

Написання тестів є необхідною частиною процесу написання програмного коду в Go. Вони допомагають виявляти помилки та забезпечують стабільність коду під час розвитку проекту.

## Як

Написання тестів у Go досить просте. Слід дотримуватися кількох правил, щоб писати ефективні та зрозумілі тести.

Наприклад, щоб створити тестовий файл у Go, необхідно:

```Go
package main

// імпортуємо пакет для тестування
import "testing"

// напишіть функцію для тестування
func TestAddition(t *testing.T) {
	// встановіть вхідні дані
	num1 := 2
	num2 := 3

	// викликайте функцію, яку потрібно протестувати
	result := num1 + num2

	// задання очікуваного результату
	expected := 5

	// порівняння фактичного та очікуваного результатів
	if result != expected {
		t.Errorf("Додавання не видало очікуваний результат. Очікувано: %d, отримано: %d", expected, result)
	}
}

// запускаємо тестування
// тест буде неуспішний, тому що ми очікуємо результат 5, а отримали 6
```

Ми написали функцію для тестування додавання і викликали її у цьому ж файлі. За допомогою *testing.T, який передається у функцію, ми можемо перевірити, чи отримали очікуваний результат. Якщо тест успішний, ви побачите "PASS", якщо ні – "FAIL". Додавання нових тестів можливе в будь-який час.

## Deep Dive

Написання тестів у Go дозволяє перевірити роботу різних функцій та методів вашого коду. Вони можуть бути корисні як при написанні нового коду, так і при зміні та вдосконаленні уже існуючого.

Важливо дотримуватися кількох правил при написанні тестів у Go:

- Починайте назви тестів з "Test" та використовуйте капіталізацію для розділення окремих слів (наприклад, "TestMyFunction").
- Використовуйте *testing.T у функціях для тестування, щоб перевірити правильність результатів.
- Використовуйте команду "go test" для запуску тестів.
- Не забувайте перевіряти розміри та типи даних у тестах.

Можна також використовувати бібліотеки для тестування, такі як "testify" або "ginkgo", які дозволяють написати читабельніші та зручніші тести.
