## Що і чому?

Читання аргументів командного рядка це, в основному, дія, в результаті якої програма отримує дані від користувача для її виконання. Програмісти це роблять для гнучкості програм, дозволяючи користувачам передавати параметри, що змінюють поведінку програми.

## Як:

```Haskell
import System.Environment

main :: IO ()
main = do
  args <- getArgs
  print args
```

Цю програму б можна викликати так: `./prog abc 123`, і вона виведе `["abc","123"]`.

## Поглиблено:

Спосіб запису аргументів командного рядка був розроблений понад 50 років тому і є основоположним елементом інтерфейсу командного рядка. У Haskell (він же GHC) з'явилася можливість читання аргументів командного рядка ще з перших версій мови. Альтернативою є зчитування параметрів з файла або через введення вручну під час виконання програми.

За замовчуванням Haskell (через функцію getArgs) повертає список об'єктів типу `String`, які представляють аргументи, передані програмі. Функция getArgs є частиною бібліотеки System.Environment і повертає всі аргументи командного рядка, крім назви самої програми (`argv[0]` у термінах C).
