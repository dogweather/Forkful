## Чому
Написання текстового файлу є важливою частиною програмування в Haskell, оскільки це дозволяє зберігати дані для подальшого використання та обробки. Крім того, це є зручним способом для збереження результатів обчислень.

## Як
Існує кілька способів створення текстового файлу в Haskell. Один з них - використання вбудованої функції `writeFile`, яка приймає два аргументи: шлях до файлу та вміст, який необхідно записати.

Простий приклад коду для створення файлу з одним рядком виглядатиме наступним чином:

```Haskell
main = do
    writeFile "my_file.txt" "Це приклад тексту, який буде записаний у файл."
```

Якщо потрібно додати більше рядків до файлу, ми можемо використати функцію `appendFile`, яка додає новий вміст у кінець існуючого файлу.

```Haskell
main = do
    appendFile "my_file.txt" "Цей рядок буде доданий після першого."
    appendFile "my_file.txt" "А цей - після другого."
```

В результаті, у документі буде виглядати наступним чином:

Це приклад тексту, який буде записаний у файл.
Цей рядок буде доданий після першого.
А цей - після другого.

## Глибоке поглиблення
При створенні та редагуванні текстових файлів важливо розуміти різні види енкодувань. Haskell пропонує декілька вбудованих функцій для роботи з різними типами енкодувань, такими як UTF-8, UTF-16, та іншими.

Наприклад, функція `writeFile` дозволяє задати енкодування як опційний параметр, якщо використовується нестандартне значення.

```Haskell
main = do
    writeFile "my_file.txt" "Цей текст буде збережений в UTF-8" (Just utf8)
```

Аналогічно, функція `appendFile` має еквівалентний параметр для задання енкодування. Детальніше про роботу з різними енкодуваннями можна дізнатися в офіційній документації по Haskell.
