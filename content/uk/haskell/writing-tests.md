## Чому

Не останнім часом тестування коду стає все більш популярним серед програмістів усього світу. А що ж з цим пов'язано?

Написання тестів допомагає підтримувати якість програмного коду на високому рівні, запобігає появі багів та полегшує роботу команди програмістів.

## Як це зробити

Написання тестів в мові програмування Haskell дуже просте та зручне. Для початку створимо новий проект за допомогою команди ```stack new project-name```, а потім встановимо бібліотеку для тестування за допомогою команди ```stack install hspec```.

Далі додамо необхідний файл для написання тестів та імпортуємо необхідні бібліотеки у ньому:

```Haskell
-- test/Spec.hs
import Test.Hspec

main :: IO ()
main = hspec $ do
  describe "Функція додавання" $ do
    it "1 + 1 повинно дорівнювати 2" $ do
      (1 + 1) `shouldBe` 2
```

Далі запустимо тест за допомогою команди ```stack test```. Якщо тест пройшов успішно, у виводі ми побачимо повідомлення "1 example, 0 failures".

## Глибоке вивчення

Написання тестів використовуючи вбудовану бібліотеку Hspec - це лише один із способів. Існують і інші бібліотеки для написання тестів у мові Haskell, наприклад, QuickCheck та doctest.

Також важливо розуміти, що тестування повинно бути охоплювати як можна більше критичних випадків та можливих помилок. Тому варто розглянути інструменти для автоматичного генерування тестових даних та перевірки коду на наявність помилок.

Також, варто пам'ятати про те, що написання тестів - це не тільки про перевірку правильності виконання коду, але й про документування функціональності програми та допомогу при подальшому розробці та зміні коду.
