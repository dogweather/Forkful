## Чому

Використання виведення дебаг-повідомлень є важливим інструментом при вирішенні проблем у програмуванні на Haskell. Воно дозволяє нам отримати більш детальну інформацію про те, що відбувається в нашому коді під час виконання програми. Це може допомогти нам знайти та виправити помилки, а також зрозуміти, як наш код працює.

## Як використовувати виведення дебаг-повідомлень?

Для того, щоб додати дебаг-повідомлення до нашого коду, ми можемо використовувати функцію `trace` з бібліотеки `Debug.Trace`. Для цього нам потрібно вказати текст, який ми хочемо вивести, а також значення, яке ми хочемо додати до повідомлення. Наприклад:

```Haskell
import Debug.Trace

-- Функція, яка додає два числа
addNumbers :: Int -> Int -> Int
addNumbers x y = trace ("Додаю " ++ show x ++ " та " ++ show y) (x + y)

-- Виклик функції
addNumbers 3 5
```

В результаті, ми отримаємо наступне виведення:

```
Додаю 3 та 5
8
```

Як бачимо, ми додали повідомлення про те, які саме числа ми додаємо, і отримали результат виконання функції. Такий підхід допомагає нам краще розуміти, як наш код працює та виявляти можливі помилки.

## Глибоке дослідження

Ви можете також використовувати функцію `traceShow` для виведення значення певної змінної або результату виразу. Також є можливість додавати більш складне повідомлення за допомогою функції `traceShowM`, яка дозволяє виводити значення виразу, що повертає монаду.

Важливо пам'ятати, що функції виведення дебаг-повідомлень повертають оригінальне значення, тому вони не змінюють поведінку нашого коду під час виконання. Це дає нам змогу використовувати ці функції навіть у виробничому коді без жодного негативного впливу на продуктивність програми.
