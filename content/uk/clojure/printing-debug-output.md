## Чому

Вітання, друзі! Сьогодні ми розглянемо важливу тему для кожного програміста - друк відладкової інформації. Бувають ситуації, коли ми не можемо зрозуміти, чому наш код не працює правильно. Тому робити друк відладкової інформації - це важливий інструмент для виявлення та усунення помилок.

## Як це зробити

Для друку відладкової інформації використовуються спеціальні функції, такі як "println" або "print". Давайте подивимося на приклад коду на мові Clojure, де ми будемо друкувати значення змінної "x":

```Clojure
(def x 5)
(println x)
```

Цей код виведе "5" в консоль. Якщо ми хочемо додати додаткову інформацію, наприклад, назву змінної, ми можемо використати цикл "str" для об'єднання різних значень:

```Clojure
(def x 5)
(println (str "Значення змінної x: " x))
```

Цей код виведе "Значення змінної x: 5". Також ми можемо використовувати умовні вирази для друкування тільки в певних ситуаціях:

```Clojure
(def x 5)
(def y 10)
(if (> x y)
    (println "Змінна y більша за змінну x")
    (println "Змінна x більша за змінну y"))
```

У цьому прикладі, якщо змінна "x" більша за "y", то буде виведено "Змінна y більша за змінну x", і навпаки - "Змінна x більша за змінну y".

## Глибоке пірнання

Друк відладкової інформації - це корисний інструмент для виявлення помилок, але його слід використовувати з розумінням. Надмірне використання друку може призвести до засмічення консолі, що ускладнить читання виводу. Також слід бути уважним при використанні друку у важких циклах або з великою кількістю даних, оскільки це може сповільнити роботу програми.
