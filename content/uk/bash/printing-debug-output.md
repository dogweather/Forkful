##Чому
В програмуванні часто зустрічаються помилки та неправильні дії. Для того, щоб зрозуміти, де саме трапився збій та як його виправити, корисно використовувати вивід дебагу. Це дозволяє бачити, що саме відбувається в коді в певній точці та вносити необхідні зміни.

##Як
Найпростіший спосіб виводу дебагу в Bash - за допомогою команди `echo`. У коді програми можна вставити цю команду та додати потрібний текст або значення змінної. Це дозволить побачити, які значення мають змінні в певній точці коду.

Наприклад:
```Bash
#!/bin/bash

# Приклад виводу значення змінної

name="Олексій"
echo "Привіт $name! Як справи?"
```

В результаті в консолі ми побачимо:
```Bash
Привіт Олексій! Як справи?
```
Також можна використовувати команду `printf` для виводу форматованого дебагу. Ця команда дозволяє вказати певний формат для виводу, наприклад, кількість знаків після коми для дійсних чисел або ширину виводу для рядка.

Наприклад:
```Bash
#!/bin/bash

# Приклад форматованого виводу

number=5.4321
printf "Число з точністю до двох знаків після коми: %.2f\n" $number
```

В результаті отримаємо:
```Bash
Число з точністю до двох знаків після коми: 5.43
```

##Глибинне дослідження
В Bash також є можливість виводити в дебаг не лише значення змінних, а й результати виконання певних команд або скриптів. Для цього можна використовувати символ `|` (pipe) та команду `tee`.

Наприклад, уявімо, що ми маємо скрипт `add.sh`, який додає два числа, та хочемо побачити результат виводу дебагу в разі помилки. Ми можемо зробити це таким чином:
```Bash
#!/bin/bash

# Скрипт для додавання двох чисел

sum=$(( $1 + $2 ))
echo "Результат: $sum" | tee -a debug.log
```
Тут ми використовуємо команду `tee` для виведення результату додавання у файл `debug.log` та на екран.
