## What & Why?
## Що і чому?
Капіталізація рядкових даних - це процес зміни першої літери у слові на велику букву. Програмісти юзають це для нормалізації даних, форматування виводу і для врахування граматичних правил мови.

## How to:
## Як це зробити:
```Bash
# Базове використання команди tr для переведення рядка у верхній регістр
echo "привіт світ" | tr '[:lower:]' '[:upper:]'
# Вивід: ПРИВІТ СВІТ

# Використання awk для капіталізації кожного слова
echo "привіт світ" | awk '{for (i=1; i<=NF; i++) $i=toupper(substr($i, 1, 1)) substr($i, 2)} 1'
# Вивід: Привіт Світ

# Скрипт Bash, що капіталізує першу літеру
capitalize() {
    local input=$1
    echo "${input^}"
}
capitalize "привіт світ"
# Output: Привіт світ
```

## Deep Dive:
## Поглиблений огляд:
У UNIX-подібних системах використання команд для обробки тексту має довгу історію, де застосування інструментів як `tr`, `awk` стандартна практика. Однак, Bash з версії 4.0 вводить власні вбудовані можливості для роботи з рядками: `${input^}` капіталізує першу літеру, а `${input^^}` перетворює весь рядок у верхній регістр.

Більшість мов програмування мають вбудовану функціональність для капіталізації, але в Bash, набори утиліт командного рядка часто легші та швидкі для простих задач. Наприклад, `tr` ефективно працює для перетворення всього рядка, тоді як `awk` є ідеальним для маніпуляцій з окремими словами.

Щодо реалізації, капіталізація слова у Bash може бути трохи менш очевидна, ніж у високорівневих мовах. Ви маєте пам'ятати, що капіталізація впливає на кодування символів; вона базується на ASCII для англійської мови, але може бути складнішою для Unicode, який використовується з безліччю інших мов.
