## Що та Чому?

Модифікація файлів за допомогою однорядкових команд CLI (інтерфейс командного рядка) полягає в здійсненні швидких, цілеспрямованих змін в файлах прямо з вашого терміналу. Програмісти роблять це тому, що це швидко, може бути автоматизовано через скрипти та, працюючи в середовищах на кшталт Linux, часто це найпростіший спосіб внести модифікації без відкриття реального редактора. Це використовує можливості sed, awk, grep та інших інструментів командного рядка для пошуку, заміни, вставки або видалення вмісту файлів на льоту.

## Як:

Давайте розглянемо кілька основних прикладів:

1. **Заміна тексту** в файлі за допомогою `sed`:
   ```Bash
   sed -i 's/oldText/newText/g' filename.txt
   ```
   Ця команда шукає `oldText` у `filename.txt` та замінює його на `newText`.

2. **Додавання тексту** до файлу:
   ```Bash
   echo "Новий рядок тексту" >> filename.txt
   ```
   Додає новий рядок тексту до кінця `filename.txt`.

3. **Видалення рядка**, що містить певний рядок з `sed`:
   ```Bash
   sed -i '/stringToDelete/d' filename.txt
   ```
   Видаляє рядки, що містять `stringToDelete` з `filename.txt`.

4. **Витягування та виведення** рядків, які відповідають шаблону за допомогою `grep`:
   ```Bash
   grep 'patternToMatch' filename.txt
   ```
   Показує рядки з `filename.txt`, які відповідають шаблону.

## Поглиблений Аналіз

Модифікація файлів за допомогою однорядкових команд CLI є технікою, старою як сам Unix, що значно покладається на інструменти на кшталт `sed`, `awk`, `grep`, та `cut`. Ці утиліти були створені на початках Unix для ефективної обробки тексту, використовуючи тоді революційну концепцію пайплайнів.

**Альтернативи**: Хоча ці однорядкові команди є потужними, вони мають обмеження, особливо при роботі з більш складними структурами даних або бінарними файлами. У таких випадках вищорівневі скриптові мови, як Python або Perl, можуть бути більш доцільними завдяки їх розширеним можливостям аналізу та маніпуляції з даними.

**Деталі реалізації**: Розуміння регулярних виразів (regex) є критичним при роботі з цими інструментами, оскільки вони є основою зіставлення шаблонів та маніпуляції з текстом. Більше того, опція `-i` з `sed` для редагування в місці не працює універсально на всіх системах однаково, особливо на macOS порівняно з Linux, де ви можете потребувати включити аргумент для розширення резервного копіювання з `-i` на macOS.
