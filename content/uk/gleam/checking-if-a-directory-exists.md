## What & Why?
Що це та навіщо?

Перевірка наявності директорії – це процес визначення, чи існує на файловій системі конкретна папка. Програмісти роблять це, щоб уникнути помилок під час читання з або запису в файли, що знаходяться у вказаній директорії.

## How to:
Як це зробити:

```gleam
// Цей код у Gleam ілюструє, як перевірити наявність директорії.
// Не забувайте, що Gleam ще зростає, і інтерфейси можуть змінюватися.

import gleam/io

pub fn check_directory_exists(dir: String) -> Bool {
  // імплементація залежить від вибраної бібліотеки або FFI
  // показано прикладний код
  io.is_dir(dir)
}

fn main() {
  let directory = "example_dir"
  let exists = check_directory_exists(directory)
  io.println(exists)
}

// Запустивши цей приклад, ви побачите `true` або `false` у виводі,
// залежно від того, чи існує 'example_dir'.
```

## Deep Dive:
Поглиблений аналіз:

Історично, перевірка наявності директорії – це завдання, яке залежало від операційної системи. У багатьох мовах програмування це робиться через системні виклики або отримання відповіді від файлової системи. У Gleam, сильно типізованій мові, яка компілюється у Erlang bytecode, ви зазвичай використовуєте бібліотеки чи функциональность інтегровану із Erlang для такої перевірки. Зараз у Gleam немає стандартної бібліотеки для роботи з файловою системою, тож може знадобитися FFI (Foreign Function Interface) або використання обгортки для Erlang функцій, таких як `filelib:is_dir/1`.

Інші спільноти, зокрема Elixir і Erlang, мають схожі виклики, що дозволяють зробити перевірку директорії, і ви можете скористатися цими варіантами у вашому Gleam проекті. Підходи можуть відрізнятися за продуктивністю та точністю; наприклад, деякі можуть не враховувати символічні посилання.
