## Що та Чому?

Генерування випадкових чисел у програмуванні може бути критично важливим для створення симуляцій, тестування, криптографії та ігор. У Gleam це функціонал, який дозволяє розробникам вносити непередбачуваність або симулювати реальні сценарії у своїх додатках.

## Як це зробити:

Для генерації випадкових чисел у Gleam ви переважно використовуєте бібліотеку `gleam_random`. Ця бібліотека надає функції для генерування випадкових цілих чисел, чисел з плаваючою комою та іншого. Перш за все, переконайтеся, що ви додали `gleam_random` до вашого файлу `rebar.config` або `mix.exs` як залежність.

Давайте розглянемо деякі приклади:

### Генерація Випадкового Цілого Числа

Щоб отримати випадкове ціле число в заданому діапазоні, ви можете використовувати функцію `int`:

```gleam
import gleam/random

pub fn generate_random_int() {
  let random_int = random.int(1, 10)
  random_int
}
```

Ця функція згенерує випадкове ціле число між 1 та 10 включно.

### Генерація Випадкового Числа з Плаваючою Комою

Щоб отримати випадкове число з плаваючою комою, використовуйте функцію `float`. Вона генерує число з плаваючою комою між 0.0 та 1.0:

```gleam
import gleam/random

pub fn generate_random_float() {
  let random_float = random.float()
  random_float
}
```

### Приклад Виводу

Виконання цих функцій може дати такі результати як:

- Для `generate_random_int()`: `5`
- Для `generate_random_float()`: `0.84372`

Пам'ятайте, що кожне виконання може привести до різних результатів через природу випадковості.

## Поглиблений Аналіз

Модуль `gleam_random` реалізує генератор псевдовипадкових чисел (PRNG), що по суті означає, що числа не є справді випадковими, але їх важко передбачити, імітуючи випадковість. PRNG працюють, починаючи з початкового значення, відомого як посів, і застосовуючи математичні операції для генерації послідовності чисел.

Історично, мови та бібліотеки реалізували кілька алгоритмів для PRNG, таких як Мерсенна Твістер або Лінійний Конгруентний Генератор (LCG). Вибір алгоритму впливає на якість "випадковості", деякі з них більше підходять для криптографічних застосувань, ніж інші. Хоча стандартна бібліотека Gleam забезпечує зручність і легкість використання зі своїм модулем `gleam_random`, вона може не завжди бути найкращим вибором для випадків, які вимагають криптографічно безпечної випадковості. Для криптографічних цілей розробники повинні шукати бібліотеки, спеціально призначені для надання криптографічно безпечних генераторів псевдовипадкових чисел (CSPRNG), які розроблені так, щоб витримати атаки, які можуть передбачити майбутні числа, спостерігаючи за послідовністю згенерованих чисел.

На завершення, хоча функціонал генерування випадкових чисел у Gleam є надійним для загальних потреб програмування, додатки з конкретними вимогами до безпеки повинні розглянути спеціалізовані криптографічні рішення, щоб забезпечити цілісність і безпеку своєї генерації випадкових чисел.
