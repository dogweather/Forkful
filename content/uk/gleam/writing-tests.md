## Чому

Написання тестів є важливою частиною процесу розробки програмного забезпечення. Тести допомагають виявляти помилки і забезпечують відповідну роботу програми. Також вони допомагають зберегти час та зусилля, оскільки ви можете перевірити, чи все працює правильно за допомогою тестів, а не вручну перевіряти кожну частину коду.

## Як

Написання тестів у Gleam є простим і ефективним процесом. Існує кілька основних кроків, які ви повинні дотримуватися, щоб написати ефективні тести:

1. Створіть тестовий файл з розширенням `.gleam`.
2. Імпортуйте модуль `testing` для використання у програмі.
3. Створіть тестову функцію з викликом модуля `testing` та назвою тесту.
4. Використовуйте `expect` для перевірки, чи провалиться тест за допомогою заданого значення або вислову.

```Gleam
import testing

fn test_addition() {
    testing.test("Правильно додає числа", {
        expect(1 + 1).toEqual(2)
    })
}
```

Запустіть ваші тести за допомогою Gleam команди `test` та перегляньте результати. Якщо ваші тести успішно пройдуть, ви побачите підтвердження у консолі. В іншому випадку, ви побачите повідомлення про те, що тест провалено, а також подробиці про помилку.

## Deep Dive

Існує багато різних функцій та методів для створення більш складних тестів у Gleam. Наприклад, ви можете використовувати методи `toEqual`, `toBeLessThan`, `toMatch`, та багато інших для перевірки різних видів даних та поведінки програми.

Також, ви можете створити багато різних тестових функцій та запускати їх разом, щоб перевірити різні аспекти програми. Це допоможе вам переконатися, що ваша програма працює як очікувалось у різних умовах.
