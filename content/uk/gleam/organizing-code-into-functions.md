## Що і чому?
Організація коду у функції означає поділ поведінки програми на менші, повторювані частини. Програмісти роблять це для того, щоб код був зрозумілішим, більш піддається обслуговуванню та запобігав повторенням.

## Як це зробити:
Ось простий приклад організації коду у функції на Gleam:

```gleam
fn add(x, y) {
  x + y
}

fn main() {
  let sum = add(3, 4)
  sum
}

// Приклад виводу
// 7
```

У цьому фрагменті `add` є функцією, яка приймає два значення і додає їх. `main` - це місце, де ми викликаємо `add` та управляємо результатом.

## Поглиблене вивчення
Історично, концепція функцій (або "підпрограм") зробила революцію в програмуванні, відкривши шлях для структурованого програмування у 1960-х роках та далі. Функції заохочують модульний підхід, де проблеми діляться на підпроблеми, вирішуються незалежно і компонуються для вирішення більшої проблеми.

У Gleam, який є мовою зі строгою типізацією, функції також несуть інформацію про типи, що гарантує їх використання у відповідності з їх означенням. Це зменшує помилки та уточнює наміри.

Альтернативи функціям включають інлайнове кодування, де логіка повторно викладається у тексті. Хоча іноді це швидше для малих, одноразових завдань, інлайнове кодування не масштабується добре для більших застосунків.

Деталі реалізації, які слід враховувати, коли ви організовуєте код у функції, можуть включати композицію функцій, де функції використовуються як будівельні блоки, та функції вищого порядку, які приймають інші функції як аргументи або повертають їх, додаючи гнучкість до того, як код організований і виконаний.
