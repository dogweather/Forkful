## Що та Чому?
Читання текстового файлу — це процес отримання даних з файла, який зберігає текст. Програмісти цим займаються, щоб обробляти інформацію, налаштовувати програми через конфігураційні файли і взаємодіяти з користувачами через лог-файли.

## Як це зробити:
У Gleam читання файлів здійснюється через модуль `gleam/io`. Нижче — простий приклад.

```gleam
import gleam/io

pub fn main() {
  // Спробуємо відкрити і прочитати файл
  case io.read_file("hello.txt") {
    Ok(contents) ->
      io.println(contents) // Якщо все добре, виведемо вміст на екран

    Error(error) ->
      io.println("Не вдалось відкрити файл: " ++ error) // Якщо є помилка, повідомимо про це
  }
}
```

Якщо у файлі "hello.txt" буде текст "Привіт, Світ!", вивід програми буде:

```
Привіт, Світ!
```

## Поглиблений аналіз
Читання файлів у программуванні — давня потреба. Спершу це було частиною операційних систем та мови C, де `fopen` та `fgets` домінували. У сучасних мовах, таких як Gleam, ми маємо більш вишукані API, які враховують асинхронність та типізацію.

Альтернативами є: використання баз даних, API чи сервісів зберігання у хмарі – але для локальних та простих завдань читання з файлу все ще незамінне.

Деталі реалізації: у Gleam із модулем `gleam/io` використовують абстракції ОС для взаємодії з файловою системою, забезпечуючи переносимість коду. Це типова практика у багатьох мовах, прагнення до того, щоб ізолювати програміста від розбіжностей систем.
