## Чому

Написання тестів є необхідною частиною процесу програмування на TypeScript. Вони допомагають впевнитися, що код виконує потрібні завдання та не має помилок, які можуть виникнути під час розвитку проекту. Написання тестів також допомагає забезпечити належну якість коду та полегшує процес розробки.

## Як

Найпростіший спосіб написати тести на TypeScript - це використовувати фреймворк для тестування, такий як Jest або Mocha. Слід визначити функції або компоненти, які потрібно протестувати та написати сценарії для перевірки їх функціональності. Давайте подивимось на приклад, як перевірити, чи правильно працює функція додавання:

```TypeScript
const add = (a: number, b: number): number => a + b;

test('додавання чисел', () => {
  expect(add(2, 3)).toEqual(5);
  expect(add(-1, 5)).toEqual(4);
});
```

У цьому прикладі ми створюємо функцію `add`, яка приймає два числа та повертає їх суму. Далі ми написали тест, що перевіряє, чи повертає ця функція правильне значення для різних вхідних параметрів. Тепер, якщо ми змінимо функцію `add` та зробимо помилку, наш тест не пройде, що дозволяє зловити помилку на ранніх етапах розробки.

## Глибоке занурення

Написання тестів - це більше, ніж просто перевірка правильного функціонування коду. Вони також допомагають уникнути деяких спільних помилок, таких як проблеми з функціональними залежностями та масштабованістю коду. Наприклад, написання тестів під час розвитку проекту з допомогою TypeScript допоможе виявити, коли інтерфейси та типи паразитують між собою, що може привести до непередбачуваних помилок.

Також, написання тестів дає змогу створювати більш гнучкий та масштабований код. Завдяки тестам можна швидко визначити, коли необхідно рефакторити частину коду для покращення функціональності або забезпечення належної якості коду.
