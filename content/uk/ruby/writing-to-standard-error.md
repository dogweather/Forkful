## What & Why? / Що та Чому?
Standard error (stderr) - це потік, призначений для виведення помилок та діагностичної інформації. Програмісти використовують його, щоб розділяти звичайний вивід програми від повідомлень про помилки, що полегшує відлагодження та аналіз логів.

## How to / Як це зробити:
```Ruby
# Прямий вивід у стандартну помилку (stderr)
$stderr.puts "Це повідомлення про помилку"

# Альтернативний спосіб використання STDERR
STDERR.puts "Ще одне повідомлення про помилку"

# Використання warn для виводу у stderr
warn "Попередження! Щось пішло не так."

# Пример перенаправлення stderr у файл
File.open('errors.txt', 'w') do |file|
  $stderr.reopen(file)
  $stderr.puts "Це повідомлення з'явиться в файлі errors.txt"
end
```

**Зразок виводу:**
```
Це повідомлення про помилку
Ще одне повідомлення про помилку
Попередження! Щось пішло не так.
```

## Deep Dive / Поглиблений аналіз:
Стандартний потік помилок (stderr) виник у Unix системах разом з концепцією потоків для виводу та вводу. Важливість stderr полягає у його використанні для логування помилок окремо від основного виводу (stdout), що дозволяє легко перенаправити ці помилки для аналізу без змішування з регулярними даними. Ruby забезпечує вбудовані глобальні змінні `$stderr` і `STDERR` для робіт із stderr. Вони можуть бути перевизначені або перенаправлені за потреби, як показано у прикладах вище.
