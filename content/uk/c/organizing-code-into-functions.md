## Що та Чому?
Організація коду у функції полягає у поділі коду на багаторазово використовувані блоки, що виконують певні завдання. Це робить код легшим до читання, відлагодження та підтримки.

## Як це зробити:
Візьмемо простий приклад: скажімо, ви хочете додати два числа декілька разів.

Без використання функцій:
```C
#include <stdio.h>

int main() {
    int sum1 = 5 + 3;
    printf("Sum1: %d\n", sum1);
    
    int sum2 = 2 + 8;
    printf("Sum2: %d\n", sum2);
    
    // Тут можуть бути додаткові додавання...
    
    return 0;
}
```

З використанням функцій:
```C
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int sum1 = add(5, 3);
    printf("Sum1: %d\n", sum1);
    
    int sum2 = add(2, 8);
    printf("Sum2: %d\n", sum2);
    
    // Використовуйте функцію add() для більшої кількості додавань...
    
    return 0;
}
```

Вивід:
```
Sum1: 8
Sum2: 10
```

## Поглиблений Розгляд
Перед тим як в C з'явилися функції, програмування часто велося лінійним чином, схоже на рецепт. Але коли програми стали більшими, дублювання коду стало проблемою. Функції були рішенням - вони дозволили нам виконувати той самий блок коду з різних частин програми без повторного написання його щоразу. Це не лише зберігає місце, але й час під час оновлень: змініть функцію в одному місці, і кожна частина вашого коду, що її використовує, отримає оновлення.

Альтернативами для функцій можуть бути вбудований код, макроси або кодування з копіюванням-вставкою, але це може призвести до надмірного, схильного до помилок та важкого до підтримки коду. Функції, у свою чергу, інкапсулюють функціональність, визначають чіткі інтерфейси і можуть зменшувати побічні ефекти за рахунок правильного використання області видимості.

Коли ви імплементуєте функції, враховуйте пару деталей: по-перше, намагайтеся зробити так, щоб вони виконували лише одне завдання – це відомо як Принцип Єдиної Відповідальності. По-друге, імена мають значення – вибирайте дескриптивні назви для функцій та їх параметрів, щоб ваш код був самодокументувальним.
