## Чому

Не зважаючи на те, що робота з тестами може бути тривіальна та часовим затратною, вона є важливою частиною процесу розробки програмного забезпечення. Написання тестів допомагає забезпечити якість коду, спрощує налагодження та зменшує кількість помилок у програмі. 

## Як

```C
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    // Оголошення тестових змінних
    int result = 0;
    int expected = 5;

    // Виклик функції та перевірка результату
    result = add(2, 3);

    // Порівняння отриманого та очікуваного результатів
	if (result == expected) {
	    printf("Тест виконаний успішно!");
	} else {
	    printf("Тест провалений. Отриманий результат: %d, очікуваний результат: %d", result, expected);
	}
    
    return 0;
}
```

В цьому прикладі показано, як можна написати простий тест для функції додавання. Спочатку визначається змінна для очікуваного результату, потім виконується функція та результат перевіряється за допомогою `if`-розгалуження. 

## Глибинний аналіз

Написання тестів може бути дещо складнішою задачею, ніж тільки перевірка результату функції. Наприклад, можна створити так званий "набір тестів", що дозволяє виконувати необхідні перевірки для кожної окремої функції або частини коду. Також можна використовувати різні бібліотеки для написання тестів та автоматизації процесу.
