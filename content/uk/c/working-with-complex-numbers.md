## Що та Чому?
Комплексні числа, що є сумішшю дійсних та уявних частин (наприклад, 3 + 4i), є ключовими в складних обчисленнях, таких як обробка сигналів або розв’язування певних рівнянь. Програмісти працюють з ними в обчислювально важких застосунках, де традиційні числа не справляються.

## Як використовувати:
З моменту C99, в C є власний тип complex та бібліотека. Ось як їх використовувати:

```C
#include <stdio.h>
#include <complex.h>

int main() {
    // Оголошення двох комплексних чисел
    double complex z1 = 1.0 + 3.0 * I;
    double complex z2 = 2.0 - 2.0 * I;

    // Операції з комплексними числами
    double complex sum = z1 + z2;
    double complex mult = z1 * z2;

    // Виведення результатів
    printf("Сума: %.1f + %.1fi\n", creal(sum), cimag(sum));
    printf("Добуток: %.1f + %.1fi\n", creal(mult), cimag(mult));

    // Абсолютне значення та кутовий коефіцієнт
    printf("Абс(z1): %f\n", cabs(z1));
    printf("Арг(z1): %f\n", carg(z1));

    return 0;
}
```

Приклад виводу:
```
Сума: 3.0 + 1.0i
Добуток: 8.0 + 2.0i
Абс(z1): 3.162278
Арг(z1): 1.249046
```
## Поглиблений Розділ
Комплексні числа мають столітню історію, починаючи з алгебри 16-го століття. Прискорюючи час, вони тепер є стандартом у багатьох мовах програмування, не лише в C.

Стандарт C99 ввів `<complex.h>`, заголовок, що визначає макроси, функції, та тип данних `complex`. Існують альтернативи - як-от створення власної структури, але чому заново винаходити колесо? Стандартна бібліотека C оптимізована та готова до використання.

Незважаючи на його переваги, підтримка комплексних чисел у C має своїх критиків. Вона може бути менш інтуітивною, ніж подібні функції в мовах на кшталт Python, та оброблення крайніх випадків може бути складним. Але з точки зору чистої продуктивності, це все ще надійний вибір.
