## Чому
Навіщо взагалі виводити результати відладки? Для багатьох програмістів цей підхід може здатися неефективним чи зайвим. Але насправді, виведення додаткової інформації може бути дуже корисним при вирішенні проблем та пошуку помилок у програмі.

## Як
Приклад коду нижче показує, як можна використовувати вивід на екран для відладки простого калькулятора. В результаті після запуску програми ми будемо бачити значення змінних та кінцевий результат обчислень.

```C++
#include <iostream>
using namespace std;

int main() {
    int a = 5;
    int b = 3;
    // Виводимо значення змінних для відладки
    cout << "Значення a: " << a << endl;
    cout << "Значення b: " << b << endl;
    int result = a + b;
    // Виводимо результат обчислень
    cout << "Результат: " << result << endl;
    return 0;
}

```

Виведення додаткової інформації також дозволяє зрозуміти, як саме працює програма та дослідити її поведінку у різних випадках. Наприклад, можна вивести значення змінних під час різних ітерацій циклу для перевірки правильності обчислень.

## Глибокий аналіз
Виведення результатів відладки дозволяє пропускати кроки програми та бачити відразу кінцевий результат. Також воно допомагає виділити ті моменти, де можуть виникнути проблеми та швидко їх виявити та виправити. Більш досвідчені програмісти часто використовують спеціальні бібліотеки для зручної відладки та виведення додаткових повідомлень у консоль.
