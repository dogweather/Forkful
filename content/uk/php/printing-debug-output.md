## Чому

В програмуванні немає нічого більш незрівнянного, ніж розуміння, чому ж код саме не працює. Але якщо ви підтримуєте свій код з виводом налагодження (debug output), ви можете більш ефективно розібратися з проблемами та відшукати баги. Тому давайте поглянемо на те, як це зробити за допомогою PHP.

## Як

Найпростішим способом друкувати відладочний вивід є використання функції `print_r` або `var_dump`. Наприклад, якщо у нас є масив `$fruits`, ми можемо викликати `print_r($fruits)` для виведення вмісту цього масиву. В результаті ми отримаємо наступний вивід:

```PHP
Array
(
    [0] => apple
    [1] => banana
    [2] => orange
)
```

Це дуже корисно, але часом ми хочемо бачити вміст змінної у конкретному місці коду. Для цього ми можемо використати функцію `echo`, яка виводитиме намічений текст у вікно браузера. Наприклад, `echo "Hello World!"` виведе "Hello World!" прямо на сторінку. 

За допомогою цих простих функцій ви можете виводити будь-який тип даних, що дозволить вам легко відстежувати стан вашої програми та знайти причини неполадок.

## Глибоке занурення

Окрім використання вбудованих функцій, ви також можете створювати власні функції для виводу налагодження. Наприклад, ви можете створити функцію `debug` з параметром `$message`, яка буде виводити ваше повідомлення з використанням `echo` та `var_dump`. Також ви можете налаштувати параметр `$level`, щоб контролювати, коли це повідомлення буде виведено. Наприклад, ви можете встановити `$level` на значення 5, і, якщо ви зміна налагодження передає менше цього значення, то повідомлення не буде виводитися.

Додатковий засіб для виводу налагодження - це використання засобів для кращого форматування. Наприклад, ви можете використовувати бібліотеку `Kint` для кращого виводу масивів та об'єктів. Крім того, ви можете налаштувати засоби для запису логів, щоб зберігати ваші повідомлення налагодження у файлі, щоб ви могли оглянути їх пізніше.
