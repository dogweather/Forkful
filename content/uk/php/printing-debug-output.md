## Що і для чого?
Виведення інформації для налагодження (debug output), це показ IT-розробникові стану програми прямо в процесі її виконання. Робимо ми це для того, щоб бачити, чи добре працює наш код, і швидше знаходити й виправляти помилки.

## Як це зробити:
```PHP
<?php
   $var = 'Hello, PHP world!';
   var_dump($var); //виведення деталей змінної
?>
```
В результаті ви отримаєте такий вихідний код:
```
string(16) "Hello, PHP world!"
```
```PHP
<?php
   error_log('This is a custom error message.'); //запис повідомлення у log-файл
?>
```
## Поглиблено:
Виведення debug-інформації не є новим підходом, його почали використовувати декілька десятиліть тому, ще до появи PHP. Цей метод є стандартом для тестування коду. За допомогою виведення debug-інформації ми можемо дізнаватися інформацію про змінні, об'єкти, масиви тощо.
Є різні методи відлагодження. Наприклад, ви можете використовувати альтернативні бібліотеки, такі як `Xdebug`, який надає більше деталей і кастомизації виведення.

## Дивіться також:
1. [PHP: var_dump](https://www.php.net/manual/en/function.var-dump.php) - документація з 'var_dump', яка містить важливу інформацію про цю функцію.
2. [PHP: error_log](https://www.php.net/manual/en/function.error-log.php) - документація з 'error_log', що показує, як записувати помилки в лог-файли.
3. [Xdebug](https://xdebug.org/) - офіційний сайт Xdebug, мощного інструменту відлагодження PHP.
