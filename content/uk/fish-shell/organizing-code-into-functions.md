## Що і чому?
Організація коду в функції полягає в угрупуванні фрагментів сценарію для виконання конкретних завдань. Ми робимо це, оскільки це робить код легшим для читання, тестування та повторного використання — нікому не хочеться пробиратися через трясовину коду-спагеті.

## Як це зробити:
У Fish ви пишете функцію з допомогою ключового слова `function`, даєте їй назву і закінчуєте `end`. Ось простий приклад:

```fish
function hello
    echo "Привіт, світ!"
end

hello
```

Вивід:
```
Привіт, світ!
```

Тепер змусимо її вітати користувача:

```fish
function greet
    set user (whoami)
    echo "Привіт, $user!"
end

greet
```

Вивід:
```
Привіт, your_username!
```

Щоб зберегти її для використання у наступних сеансах, використовуйте `funcsave greet`.

## Поглиблений огляд
Функції Fish Shell подібні до міні-сценаріїв — ви можете покласти туди майже все, що завгодно. Історично концепція функцій у скриптінгу командної оболонки зекономила безліч годин повторюваного набору та відладки. На відміну від мов програмування, таких як Python, функції оболонки більше стосуються зручності, аніж структури.

Деякі оболонки, як Bash, використовують `function` або просто фігурні дужки. Fish тримається за `function ... end` — чітко і зрозуміло. Усередині функцій Fish ви отримуєте всі переваги: параметри, локальні змінні з використанням `set -l`, і ви навіть можете визначити функцію всередині іншої функції.

Вам не знадобиться значення `return`, оскільки Fish не зацикленний на цьому; вихідні дані вашої функції є її поверненням. І якщо ви хочете мати постійні функції, доступні для майбутніх сеансів, пам'ятайте про `funcsave`.
