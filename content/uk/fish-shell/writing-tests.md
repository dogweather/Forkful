## Чому

Написання тестів є невід'ємною частиною якісного розроблення програмного забезпечення. Тести допомагають виявляти помилки та забезпечувати стабільність коду. Це підвищує якість продукту та зменшує шанси на виявлення проблем у майбутньому.

## Як

```Fish Shell``` - це потужна командна оболонка з великою кількістю можливостей. Для написання тестів існує спеціальний розширений пакет для Fish Shell - ```bass```. Встановіть його за допомогою наступної команди:

```
$ bass source <(curl -s https://raw.githubusercontent.com/edc/bass/master/setup.bash)
```

Далі можна написати свої власні тести, використовуючи звичайні команди Fish Shell та спеціальну команду для тестування ```assert```.

Наприклад, створіть файл з назвою ```example-test.fish``` та додайте такий код:

```
assert "Hello" "Hello" "Рядки мають бути однаковими"
assert true "[ -d example-directory ]" "Каталог має бути існуючим"
```

Потім запустіть тести за допомогою наступної команди:

```
$ ./example-test.fish
```

Якщо всі тести пройшли успішно, то виведеться повідомлення:

```
Тести пройшли успішно!
```

В іншому випадку, буде виведено повідомлення про помилку та інформацію про те, який тест не пройшов успішно.

## Глибоке дослідження

Тести можуть бути написані для будь-яких функцій, команд або скриптів. Також можна використовувати змінні та перевіряти результати виконання команд. Більш детальну інформацію про написання тестів можна знайти в офіційній документації Fish Shell та на сторінці проекту ```bass```.
