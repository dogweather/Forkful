## Чому

Написання тестів є важливою частиною написання програмного коду, оскільки воно дозволяє перевіряти правильність роботи програми і зменшує кількість помилок та багів. Особливо це важливо для великих і складних проектів, де вручну перевіряти кожну частину коду стає неможливо. Також тестування допомагає зрозуміти структуру програми та зберегти її роботу при внесенні змін.

## Як

Написання тестів можна здійснити за допомогою спеціальних фреймворків для тестування, таких як JUnit чи TestNG. Для початку, необхідно розбиратися з синтаксисом і основами мови Kotlin. Ви можете додати анотацію ```@Test``` перед будь-яким методом, який ви хочете тестувати. Наприклад:

```Kotlin
fun sum(a: Int, b: Int): Int {
    return a + b
}

@Test
fun testSum() {
    assert(sum(2, 3) == 5)
}
```

У цьому прикладі ми тестуємо функцію ```sum```, переконуємося, що вона правильно обчислює суму двох чисел. Якщо тест повертає помилку, це означає, що функція не працює правильно і потребує виправлення.

## Deep Dive

Коли ви розумієте основи написання тестів на мові Kotlin, ви можете вдосконалити їх, використовуючи різні методи для перевірки різних умов. Наприклад, для перевірки чи список містить певний елемент, ви можете використовувати метод ```contains```. Також важливо тестувати не тільки правильну роботу коду, але й неправильну - для виявлення і виправлення важливих помилок.
