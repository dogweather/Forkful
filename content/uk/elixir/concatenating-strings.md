## Що це та навіщо?
Конкатенація строк – це процес їх об'єднання в одну довгу строку. Програмісти використовують це для створення динамічних повідомлень, генерації унікальних ключів, чи просто для з'єднання даних, що надходять з різних джерел.

## Як це зробити:
```elixir
# Класичний спосіб
full_string = "Привіт, " <> "світе!"
IO.puts(full_string) 
# Вивід: Привіт, світе!

# Інтерполяція строк
name = "Олексій"
welcome_message = "Ласкаво просимо, #{name}!"
IO.puts(welcome_message)
# Вивід: Ласкаво просимо, Олексій!

# Використання функції `String.concat`
greeting = String.concat("Доброго ", "ранку!")
IO.puts(greeting)
# Вивід: Доброго ранку!
```

## Поглиблений розгляд
У мові Elixir, яка з'явилася у 2011 році, конкатенація строк є основною операцією. В офіційній документації рекомендується використовувати `<>`, бінарний оператор для ефективного з'єднання бітстрінгів. Інтерполяція строк швидша за конкатенацію за допомогою `String.concat`, оскільки компілятор виконує оптимізацію під час генерації байт-коду.

Альтернативою може бути використання списків строк для побудови складних текстових блоків, з подальшим викликом `Enum.join/1` на потрібному місці. Однак це варіант рідко використовується через зайву складність та погіршення читабельності коду.

В Elixir строка є бінарним типом, тобто послідовністю байтів. Це відмінне рішення з погляду ефективності, оскільки воно добре підходить для роботи з великими та/або бінарними даними.
