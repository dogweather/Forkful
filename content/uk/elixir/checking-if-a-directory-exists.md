## Що і чому?

Перевірка наявності директорії - це процес визначення, чи існує певна папка в файловій системі. Програмісти роблять це, щоб запобігти помилкам при спробі читання чи запису в неіснуючу директорію.

## Як це робити:

В Elixir перевірка наявності директорії може бути виконана за допомогою функції `File.dir?/1`. Переглянемо приклад:

```Elixir
if File.dir?("some_directory") do
  IO.puts "Директорія існує"
else
  IO.puts "Директорія не існує"
end
```

Якщо директорія існує, програма виведе "Директорія існує". В протилежному випадку, ви побачите "Директорія не існує".

## Поглиблений огляд

Elixir - молода мова, впроваджена в 2011 році, що використовує ерлангову віртуальну машину для обробки високого рівня одночасності в розподілених системах.

Альтернатива функції `File.dir?/1` - це `:file.read_file_info/1`. Ця функція повертає кортеж з інформацією про файл або директорію. Якщо переданий шлях не існує, вона поверне `{:error, :enoent}`.

```Elixir
case :file.read_file_info("some_directory") do
	{:ok, _} -> 
		IO.puts "Директорія існує"
	{:error, :enoent} ->
		IO.puts "Директорія не існує"
	_ -> 	
		IO.puts "Не вдалось перевірити директорію"
end
```

Функція `File.dir?/1` насправді використовує цю функцію в своїй реалізації, що робить її більш зручною для використання при перевірці наявності директорії.
