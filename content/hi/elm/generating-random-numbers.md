## क्या और क्यों?
Elm में यादृच्छिक संख्याएं उत्पन्न करना अनिश्चितापूर्ण संख्यात्मक मूल्यों को बनाने की प्रक्रिया है जो खेल, सिमुलेशन, और सुरक्षा एल्गोरिदम जैसे अनुप्रयोगों के लिए अनिवार्य होती है। प्रोग्रामर वास्तविक दुनिया की विविधता का अनुकरण करने, उपयोगकर्ता अनुभव को बेहतर बनाने, या सुरक्षा तकनीकों के साथ डेटा को सुरक्षित करने के लिए यादृच्छिकता का उपयोग करते हैं।

## कैसे:
Elm, कई प्रोग्रामिंग भाषाओं की तुलना में यादृच्छिकता को अलग तरीके से संभालता है, एक प्रणाली का उपयोग करते हुए जो फ़ंक्शंस को शुद्ध बनाए रखता है। यादृच्छिक संख्या उत्पन्न करने के लिए, आपको Elm के `Random` मॉड्यूल के साथ काम करना होगा। यहाँ 1 से 100 के बीच यादृच्छिक संख्या उत्पन्न करने का एक मूल उदाहरण है:

```Elm
import Html exposing (Html, text)
import Random

main : Html msg
main =
    Random.generate NewRandomNumber (Random.int 1 100)
    |> Html.map (text << toString)

type Msg = NewRandomNumber Int
```

यह छोटे कोड का टुकड़ा `Random.generate` का उपयोग करता है ताकि एक कमांड बनाई जा सके जो, जब निष्पादित किया जाता है, तो निर्दिष्ट सीमा के भीतर एक यादृच्छिक संख्या उत्पन्न करती है। `type Msg` घोषणा का उपयोग आपके Elm अनुप्रयोग के अपडेट फंक्शन में उत्पन्न संख्या को संभालने के लिए किया जाता है।

एक और इंटरेक्टिव उदाहरण के लिए, आइए एक परिदृश्य पर नज़र डालते हैं जहाँ उपयोगकर्ता क्लिक के माध्यम से यादृच्छिक संख्या उत्पन्न करने को ट्रिगर करते हैं:

```Elm
import Html exposing (Html, button, div, text)
import Html.Events exposing (onClick)
import Random

type alias Model = Int

type Msg = Generate

update : Msg -> Model -> (Model, Cmd Msg)
update msg model =
    case msg of
        Generate ->
            (model, Random.generate NewRandomNumber (Random.int 1 100))

view : Model -> Html Msg
view model =
    div []
        [ text ("Generated number: " ++ String.fromInt model)
        , button [ onClick Generate ] [ text "Generate new number" ]
        ]

type Msg = NewRandomNumber Int
```

यह Elm अनुप्रयोग इंटरेक्टिविटी को पेश करता है, हर बार उपयोगकर्ता बटन पर क्लिक करता है तो डिस्प्ले को एक नई यादृच्छिक संख्या से अपडेट करता है।

## गहराई में जानकारी
Elm की यादृच्छिक संख्या उत्पन्न करने वाली प्रणाली का डिज़ाइन भाषा की शुद्धता और पूर्वानुमानितता के प्रति प्रतिबद्धता से आता है। प्रत्येक कॉल पर विभिन्न मूल्यों को वापस करने वाले सीधे, अशुद्ध फंक्शनों के बजाय, Elm `Cmd` संरचना में यादृच्छिकता को समाहित करता है, जो शुद्ध फ़ंक्शनों से साइड इफेक्ट्स को अलग करने वाली इसकी वास्तुकला के अनुरूप है।

जबकि यह दृष्टिकोण अनुप्रयोग व्यवहार में सुसंगतता की गारंटी देता है और डिबगिंग को सुविधाजनक बनाता है, यादृच्छिक संख्याओं की अनिवार्य उत्पादन के लिए आदी लोगो�
