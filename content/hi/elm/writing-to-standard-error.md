## क्यों

हर एक प्रोग्रामर को किसी न किसी समय पर एरर मैसेज को लिखने की जरूरत पड़ती है। इससे हम अपने कोड में से गलतियों को ढूँढ़ सकते हैं और उन्हें सुधार सकते हैं। ऐसा करने से हमारा कोड और भी अच्छा बनता है और इससे हमारे प्रोग्रामिंग स्किल्स भी सुधरते हैं।

## कैसे करें

जब हम कोड में कोई गलती ढूँढ़ने के लिए `Debug.log` का उपयोग करते हैं, तो वह एक साधारण स्ट्रिंग को लिखता है। लेकिन अगर हम इसे स्टैंडर्ड एरर में लिखें तो हम उसे कहीं अन्य भी लेख सकते हैं और उसे और भी अधिक उपयोगी बना सकते हैं। इसके लिए हम निम्नलिखित कोड का उपयोग कर सकते हैं:

```Elm
import Debug
import Basics exposing (Int)

sampleFunction : Int -> Int
sampleFunction x =
  if x < 0 then
    Debug.log "Error: input should be a positive integer" 0
  else
    x + 1
```

यहां, `Int` आउटपुट का टाइप है और हम `Debug.log` का उपयोग करके उसे लिख रहे हैं। यदि `sampleFunction` को अगर कोई नेगेटिव इन्पुट पास किया जाता है, तो वह एक स्ट्रिंग और 0 रिटर्न करेगा। इसी तरह, हम अपने कोड में और भी अधिक साधारण वस्तुओं को लिख सकते हैं जिन्हें हम खोज सकते हैं और उन्हें सुधार सकते हैं।
