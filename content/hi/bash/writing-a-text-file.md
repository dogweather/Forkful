## What & Why? (क्या और क्यों?)
लिखित टेक्स्ट फाइलें डेटा भंडारण का एक साधारण रूप हैं। प्रोग्रामर लॉग्स, कॉन्फ़िगरेशन, और डेटा एनालिसिस के लिए फाइलें लिखते हैं।

## How to: (कैसे करें:)
Bash में टेक्स्ट फाइल बनाने और उसमें लिखने का आदान-प्रदान:

```Bash
# एक नई फाइल बनाना और उसमें लिखना
echo 'नमस्ते, यह मेरी पहली फाइल है!' > मेरी_फाइल.txt

# मौजूदा फाइल में जोड़ना
echo 'यह दूसरी पंक्ति है।' >> मेरी_फाइल.txt

# फाइल की सामग्री देखना
cat मेरी_फाइल.txt
```

सैंपल आउटपुट:
```
नमस्ते, यह मेरी पहली फाइल है!
यह दूसरी पंक्ति है।
```

## Deep Dive (गहराई से समझें):
टेक्स्ट फाइलों का प्रयोग Unix समय से हो रहा है, जब डेटा और कॉन्फ़िगरेशन सरल टेक्स्ट के रूप में लिखे जाते थे। विकल्पों में डेटाबेस, एक्सएमएल, और जेसन जैसे फॉर्मेट्स शामिल हैं, पर Bash में शीघ्रता से टेक्स्ट फाइल लिखना अभी भी प्रचलित है। इसे `>` और `>>` ऑपरेटरों से लागू किया जाता है, जहाँ `>` पूरे फाइल को बदल देता है, और `>>` मौजूदा सामग्री में जोड़ देता है।
