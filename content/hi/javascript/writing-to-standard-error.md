# क्यों

क्या आपने कभी कोड लिखते समय किसी गलती को ढूंढने में समस्या होने पर ध्यान दिया है? कुछ बग्स या फिर कोड चलाने के समय अनुमति त्रुटियों के कारण, आपका कोड सही तरीके से काम नहीं कर सकता है। इस समस्‍या को हल करने का एक अच्छा तरीका है इसमें लिखने के लिए standard error का उपयोग करना। इस ब्लॉग पोस्ट में हम आपको बताएंगे कि standard error क्या है और आप इसका उपयोग कैसे कर सकते हैं।

# कैसे करे

```
यह उदाहरण एक सरल Javascript कोड है जहां हमारा केस मैन कोड करता है।

var num1 = 5;
var num2 = 10;
var result = num1 + num2;

console.log(result);
```

इस कोड में हमने दो संख्याओं का जोड़ किया है और उसका फाइनल रिजल्ट डिस्प्ले करवाया है। अब अगर हमारी कोड में कोई ग़लत त्रुटि हो जाती है, तो यह डिस्प्ले करेगा:

```
NaN
```
NaN का अर्थ "Not a Number" है और यह उस ग़लत त्रुटि का संकेत देता है जो हमारे कोड में है। इससे हम आसानी से ग़लती का पता लगा सकते हैं और उसे सही कर सकते हैं।

# गहराई में

Standard error को पूछ ना हो तो यह कितनी गहराई में आता है। हमारे उदाहरण में, कोड में ग़लत त्रुटि होने के क़ारण नामुल किसी यूज़र को पता नहीं चलेगा कि उनका बनाया ग़णित पद्धति ग़लत है या नहीं। इस समस्या को हल करने का सबसे अच्छा तरीका है उस ग़लत त्रुटि को स्क्रीन पर प्रिंट करना
