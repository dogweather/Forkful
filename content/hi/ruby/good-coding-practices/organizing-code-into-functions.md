## कैसे:
कल्पना कीजिए आप उपयोगकर्ताओं का अभिवादन करने के लिए एक त्वरित स्क्रिप्ट लिख रहे हैं:

```Ruby
def greet(name)
  "Hello, #{name}!"
end

puts greet("Alice")   # आउटपुट: Hello, Alice!
puts greet("Bob")     # आउटपुट: Hello, Bob!
```

या शायद आप एक वृत्त का क्षेत्रफल मापने के लिए अंकगणित कर रहे हैं:

```Ruby
def circle_area(radius)
  Math::PI * radius ** 2
end

puts circle_area(5)   # आउटपुट: 78.53981633974483
```

साफ़ और संभालने में आसान, है ना?

## गहराई से:
रूबी में, फ़ंक्शन्स की अवधारणा, जिसे मेथड्स के रूप में भी जाना जाता है, नई नहीं है—यह प्रोग्रामिंग के समय से जितनी पुरानी है। 1950 के दशक में वापस जाने पर, सबरूटीन्स, जैसा कि वे जाने जाते थे, त्रुटिरहितता को कम करने के लिए परिचय किए गए थे।

विकल्प? बेशक, आपके पास इनलाइन कोड हो सकता है, आप ओओपी के साथ क्लासेज और ऑब्जेक्ट्स का इस्तेमाल कर सकते हैं, या यहां तक कि लैम्ब्डास और प्रोक्स के साथ फंक्शनल भी जा सकते हैं। लेकिन फ़ंक्शन्स व्यवस्थित कोड की बुनियादी बात हैं। प्रदर्शन चाहिए? फंक्शनों में स्थानीय वेरिएबल तेज होते हैं और फंक्शन `return` के साथ तत्काल मूल्य लौटा सकते हैं।

कार्यान्वयन-वार, आप `def` के साथ एक फ़ंक्शन को परिभाषित कर सकते हैं और इसे `end` के साथ समाप्त कर सकते हैं। आप डिफ़ॉल्ट पैरामीटर सेट कर सकते हैं, विविधिक फ़ंक्शन्स के लिए स्प्लैट ऑपरेटर का उपयोग कर सकते हैं, और अधिक। फंक्शन्स आपकी इच्छानुसार सरल या जटिल हो सकते हैं।
