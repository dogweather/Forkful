## क्या और क्यों?
जटिल संख्याएँ, वास्तविक और काल्पनिक भागों का मिश्रण (जैसे 3 + 4i), सिग्नल प्रोसेसिंग या कुछ समीकरणों को हल करने जैसे उन्नत गणनाओं में महत्वपूर्ण हैं। प्रोग्रामर उनसे गणित भारी अनुप्रयोगों के लिए निपटते हैं, जहाँ पारंपरिक संख्याएँ पर्याप्त नहीं होती हैं।

## कैसे:
C, C99 से, एक मूल जटिल प्रकार और पुस्तकालय है। यहाँ इसका उपयोग कैसे करें:

```C
#include <stdio.h>
#include <complex.h>

int main() {
    // दो जटिल संख्याएँ घोषित करें
    double complex z1 = 1.0 + 3.0 * I;
    double complex z2 = 2.0 - 2.0 * I;

    // जटिल संख्याओं के साथ संचालन
    double complex sum = z1 + z2;
    double complex mult = z1 * z2;

    // परिणामों की प्रिंटिंग
    printf("Sum: %.1f + %.1fi\n", creal(sum), cimag(sum));
    printf("Product: %.1f + %.1fi\n", creal(mult), cimag(mult));

    // सम्पूर्ण मान और चरण कोण
    printf("Abs(z1): %f\n", cabs(z1));
    printf("Arg(z1): %f\n", carg(z1));

    return 0;
}
```

नमूना आउटपुट:
```
Sum: 3.0 + 1.0i
Product: 8.0 + 2.0i
Abs(z1): 3.162278
Arg(z1): 1.249046
```
## गहराई से अध्ययन
जटिल संख्याएँ सदियों पुरानी हैं, जिनकी जड़ें 16वीं शताब्दी की बीजगणित में हैं। आगे बढ़ते हुए, अब वे कई प्रोग्रामिंग भाषाओं में, सिर्फ C में ही नहीं, एक मूल तत्व हैं।

C99 मानक ने `<complex.h>` को परिचय दिया, एक हैडर जो मैक्रो, फ़ंक्शन्स, और `complex` डेटा प्रकार को परिभाषित करता है। विकल्प मौजूद हैं - जैसे अपनी संरचना बनाना, लेकिन पहिए को पुनः आविष्कार क्यों करें? C मानक पुस्तकालय अनुकूलित और तत्पर है।

इसकी शक्ति के बावजूद, C का जटिल समर्थन बिना आलोचना के नहीं है। यह Python जैसी भाषाओं में समान विशेषताओं की तुलना में कम सहज हो सकता है, और कोने के मामलों को संभालना कठिन हो सकता है। लेकिन कच्चे प्रदर्शन के लिए, यह अभी भी एक मजबूत विकल्प है।
