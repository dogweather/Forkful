## क्या और क्यों?
C में त्रुटियों को संभालना अनपेक्षित की उम्मीद करने के बारे में है। यह कार्यक्रमों को बेतरतीब होने से बचाता है जब वे समस्याओं में टकराते हैं। प्रोग्रामर इसे अनुग्रह के साथ गलतियों को संभालने और अपने कोड को विश्वसनीय रखने के लिए करते हैं।

## कैसे करें:

आइए देखें C में यह कैसे करते हैं:

```C
#include <stdio.h>
#include <stdlib.h>
#include <errno.h>

int main() {
    FILE *fp = fopen("nonexistentfile.txt", "r");
    if (fp == NULL) {
        perror("फ़ाइल खोलने में त्रुटि");
        return EXIT_FAILURE;
    }
    // फ़ाइल के साथ कुछ करें
    fclose(fp);
    return EXIT_SUCCESS;
}
```

जब फ़ाइल मौजूद नहीं होती तो नमूना आउटपुट:
```
फ़ाइल खोलने में त्रुटि: ऐसी कोई फ़ाइल या निर्देशिका नहीं
```

## विस्तृत अध्ययन

C के शुरुआती दिनों में, त्रुटि संचालन काफी साधारण था - ज़्यादातर लौटाने वाले कोड्स और मैनुअल चेक्स। फिर `errno` पेश किया गया, एक वैश्विक चर जो तब अपडेट होता है जब कार्य विफल होते हैं। यह अपने आप में थ्रेड-सेफ नहीं है, हालांकि, इसलिए बेहतर त्रुटि रिपोर्टिंग के लिए नवीन `strerror` और `perror` कार्य पेश किए गए थे।

विकल्प? आधुनिक C `errno` तक सीमित नहीं है। जब आपदा आती है तो गैर-स्थानीय जम्प्स के लिए setjmp और longjmp हैं। कुछ लोग अपने त्रुटि कोड्स को परिभाषित करना पसंद करते हैं, जबकि अन्य लोग C++ में अपवाद-जैसी संरचनाओं का विकल्प चुनते हैं।

कार्यान्वयन का विवरण गहन हो सकता है। उदाहरण के लिए, `errno` POSIX अनुरूप प्रणालियों में Thread Local Storage (TLS) के जादू की वजह से थ्रेड-सेफ है। एंबेडेड सिस्टमें में, जहां संसाधन मूल्यवान होते हैं, मानक दृष्टिकोणों के ऊपर कस्टम त्रुटि संचालन कोड पसंद किया जा सकता है जो सॉफ़्टवेयर को भारी बना सकता है।
