## क्या और क्यों?
संख्याओं को गोल करना एक निश्चित स्थान के परे अंको को हटाना और वैकल्पिक रूप से अंतिम रखे गए अंक को समायोजित करना है। प्रोग्रामर सटीक मूल्यों की आवश्यकता नहीं होने पर सटीकता को कम करने के लिए, फ्लोटिंग-पॉइंट त्रुटियों को प्रबंधित करने के लिए, या उपयोगकर्ता-अनुकूल प्रदर्शन के लिए संख्याओं को तैयार करने के लिए गोल करते हैं।

## कैसे:
C में, आप आमतौर पर `floor()`, `ceil()`, या `round()` फ़ंक्शन्स का उपयोग करेंगे। यहाँ त्वरित प्रदर्शन है:

```C
#include <stdio.h>
#include <math.h>

int main() {
    double num = 3.14159;
    double num_floor = floor(num);
    double num_ceil = ceil(num);
    double num_round = round(num);

    printf("Floor: %.2f\n", num_floor); // Floor: 3.00
    printf("Ceil: %.2f\n", num_ceil);   // Ceil: 4.00
    printf("Round: %.2f\n", num_round); // Round: 3.00
    return 0;
}
```

विशिष्ट स्थान तक गोल करने के लिए अधिक नियंत्रण के लिए, आप गुणा करें, गोल करें और विभाजित करें:

```C
double roundToPlace(double num, int place) {
    double scale = pow(10.0, place);
    return round(num * scale) / scale;
}

// ...

double num = 3.14159;
double num_rounded = roundToPlace(num, 2);
printf("2 दशमलव स्थानों तक गोल किया गया: %.2f\n", num_rounded); // 2 दशमलव स्थानों तक गोल किया गया: 3.14
```

## गहरी खोज
पहले के दिनों में, संख्याओं को गोल करना अक्सर एक मैनुअल प्रक्रिया थी - सिर्फ पेन और पेपर के साथ एक भारी उठाना। कंप्यूटिंग के साथ, हमने इसे स्वचालित कर दिया, लेकिन फ्लोटिंग-पॉइंट अंकगणित ने अपनी बाइनरी प्रकृति के कारण बारीकियों को लाया, जहाँ कुछ संख्याएँ बिल्कुल भी प्रस्तुत नहीं की जा सकतीं हैं।

मानक गोलाई के विकल्पों में ट्रंकेशन (सिर्फ अतिरिक्त अंकों को ड्रॉप करना) या बैंकर्स गोलाई शामिल है, जो दो मूल्यों के बीच बिल्कुल बीच में होने पर सबसे नजदीकी सम मूल्य को गोल करता है, लगातार गणनाओं में पूर्वाग्रह को कम करता है।

मनमानी सटीकता वाली संख्याओं को गोल करने या अनंतता, संकेत नैन, या उपसामान्य मूल्यों जैसे विशेष मामलों को संभालने के लिए कार्यान्वयन जटिल हो जाता है। C स्टैंडर्ड लाइब्रेरी फ़ंक्शन्स मूल बातों को संभालते हैं, लेकिन यदि आपको कस्टम तरीकों से दशमलव को गोल करने की आवश्यकता है, तो आपको `math.h` से अधिक की आवश्यकता होगी।
