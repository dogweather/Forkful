## क्या और क्यों?
लॉगिंग मूल रूप से यह लिखने की प्रक्रिया है कि आपका प्रोग्राम क्या कर रहा है, प्रायः फ़ाइल या टर्मिनल में संदेश लिखकर। प्रोग्रामर इसे घटनाओं का ट्रैक रखने, समस्याओं की निदान करने, और समय के साथ एप्लिकेशन के संचालन की कहानी बताने वाले ऑडिट ट्रेल के लिए करते हैं।

## कैसे करें:
चलिए कुछ मूलभूत बातों से शुरुआत करते हैं। C में बिल्ट-इन लॉगिंग फ्रेमवर्क नहीं है, पर आप `stdio.h` के साथ कुछ सरल बना सकते हैं। यहाँ कैसे किया जा सकता है:

```c
#include <stdio.h>
#include <time.h>

void logMessage(const char* message) {
    time_t now;
    time(&now);
    char *date = ctime(&now);
    date[strlen(date) - 1] = '\0'; // ctime() के परिणाम के अंत में न्यूलाइन को हटा दें
    printf("[%s] %s\n", date, message);
}

int main() {
    logMessage("Application has started.");
    // ... आपका कोड यहाँ जाए ...
    logMessage("Application is doing something important.");
    // ... आपका कोड जारी है ...
    logMessage("Application has ended.");
    return 0;
}
```

नमूना आउटपुट कुछ इस तरह दिख सकता है:

```
[Tue Mar 9 12:00:01 2023] Application has started.
[Tue Mar 9 12:00:02 2023] Application is doing something important.
[Tue Mar 9 12:00:03 2023] Application has ended.
```

बेशक, असली दुनिया में आप शायद टर्मिनल के बजाय एक फ़ाइल में लिखना चाहेंगे, विभिन्न लॉग स्तरों को संभालना, और शायद एक पूर्वनिर्धारित लाइब्रेरी का उपयोग करना होगा।

## गहन अध्ययन
C में लॉगिंग का एक अजीब सा आकर्षण है—यह भाषा के अधिकांश हिस्से की तरह कम-स्तरीय है। ऐतिहासिक रूप से, `stderr` या फ़ाइल पॉइंटर के साथ `fprintf` का उपयोग करके लॉगिंग की गई थी। जैसे-जैसे प्रोग्राम जटिल हुए, वैसे-वैसे लॉगिंग की आवश्यकताएं भी बढ़ीं, जिससे सिस्टम पर `syslog` जैसे लाइब्रेरियों का विकास हुआ, जो महत्वपूर्णता के विभिन्न स्तरों के साथ अनेक स्रोतों से लॉगिंग को संभाल सकते थे।

आधुनिक परिदृश्य में, ऐसे कई C लॉगिंग लाइब्रेरियां हैं, जैसे कि `zlog`, `log4c`, और `glog`, जो लॉग रोटेशन, संरचित लॉगिंग, और मल्टीथ्रेडेड लॉगिंग सहित एक समृद्ध सुविधा सेट प्रदान करते हैं। ये समाधान लॉग शब्दावली, गंतव्यों, और प्रारूपों पर स्पष्टता से नियंत्रण रखने की अनुमति देते हैं।

लॉगिंग सिस्टम को लागू करते समय, टाइमस्टैम्प प्रारूपण, लॉग फ़ाइल प्रबंधन, और प्रदर्शन जैसे विवरणों पर विचार करने की ज़रूरत होती है। घटनाओं के मिलान के लिए टाइमस्टैम्प लॉग महत्वपूर्ण हैं, जबकि लॉग रोटेशन सुनिश्चित करता है कि लॉग फ़ाइलें बहुत अधिक डिस्क स्पेस का उपभोग न करें। लॉगिंग की प्रक्रिया भी तेज और मुख्य एप्लिकेशन प्रवाह को ब्लॉक नहीं करने वाली होनी चाहिए ताकि लॉगिंग एक बाधा न बने।

## अन्य देखें
C में लॉगिंग लाइब्रेरियों और प्रथाओं के बारे में और गहराई से जानने के लिए, इन संसाधनों को देखें:

- GNU `syslog` मैनुअल: https://www.gnu.org/software/libc/manual/html_node/Syslog.html
- `zlog`: C के लिए एक अत्यधिक कॉन्फ़िगरेबल लॉगिंग लाइब्रेरी - https://github.com/HardySimpson/zlog
- `log4c`: Log4j के अधार पर बनाया गया C के लिए एक लॉगिंग फ्रेमवर्क - http://log4c.sourceforge.net/
- `glog`: Google का एप्लिकेशन-स्तरीय लॉगिंग लाइब्रेरी - https://github.com/google/glog
