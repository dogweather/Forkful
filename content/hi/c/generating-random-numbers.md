## क्या और क्यों?

C में यादृच्छिक संख्याओं का निर्माण किसी भी स्पष्ट पैटर्न के अभाव में संख्याओं की शृंखला बनाने की प्रक्रिया है, जो यादृच्छिकता की अवधारणा की नक़ल करता है। प्रोग्रामर विविध प्रयोजनों के लिए यादृच्छिक संख्याओं का उपयोग करते हैं, जिसमें डेटा का अनुकरण, क्रिप्टोग्राफिक अनुप्रयोग, और गेम विकास शामिल हैं, जिससे यह प्रोग्रामिंग का एक महत्वपूर्ण पहलू बन जाता है।

## कैसे करें:

C में यादृच्छिक संख्या उत्पन्न करने के लिए, आप आमतौर पर `stdlib.h` में पाए जाने वाले `rand()` फ़ंक्शन का उपयोग करते हैं। हालांकि, यादृच्छिक संख्या जनरेटर को बीज देना विभिन्न प्रोग्राम निष्पादनों में उत्पन्न संख्याओं में विविधता सुनिश्चित करने के लिए महत्वपूर्ण है। एक मान के साथ बीजित `srand()` फ़ंक्शन, अक्सर वर्तमान समय, इसमें सहायता करता है।

यहाँ 0 से 99 के बीच एक यादृच्छिक संख्या उत्पन्न करने का एक सरल उदाहरण है:

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

int main() {
    // यादृच्छिक संख्या जनरेटर को बीज दें
    srand((unsigned) time(NULL));

    // 0 से 99 के बीच एक यादृच्छिक संख्या उत्पन्न करें
    int randomNumber = rand() % 100;

    // यादृच्छिक संख्या प्रिंट करें
    printf("Random Number: %d\n", randomNumber);

    return 0;
}
```

नमूना आउटपुट:

```
Random Number: 42
```

ध्यान देने की बात है कि इस प्रोग्राम की प्रत्येक निष्पादन वर्तमान समय के साथ बीजित होने के कारण एक नई यादृच्छिक संख्या उत्पन्न करेगी।

## गहराई में जानकारी

C में यादृच्छिक संख्याएँ उत्पन्न करने का पारंपरिक तरीका, `rand()` और `srand()` का उपयोग करना, वास्तव में यादृच्छिक नहीं है। यह पस्यूडोयादृच्छिक है। यह कई अन

ुप्रयोगों के लिए ठीक है, लेकिन उच्च डिग्री की यादृच्छिकता की आवश्यकता वाली स्थितियों में, जैसे कि गंभीर क्रिप्टोग्राफिक उपयोगों में, यह कम पड़ जाता है। `rand()` द्वारा उत्पन्न अनुक्रम पूरी तरह से उस  बीज पर निर्भर करता है जो `srand()` को प्रदान किया गया होता है। इस प्रकार, यदि बीज जाना जाता है, तो अनुक्रम को पूर्वानुमानित किया जा सकता है, जिससे यादृच्छिकता कम हो जाती है।

ऐतिहासिक रूप से, `rand()` फ़ंक्शन को इसकी निम्न-गुणवत्ता वाली यादृच्छिकता और सीमित रेंज के लिए आलोचना की गई है। अधिक वास्तविक यादृच्छिकता का बेहतर अनुमान लगाने वाले डिवाइस-विशिष्ट एपीआई या बाह्य लाइब्रेरीज का उपयोग करने सहित आधुनिक विकल्प शामिल हैं, या UNIX-जैसी प्रणालियों में, क्रिप्टोग्राफिक प्रयोजनों के लिए `/dev/random` या `/dev/urandom` से पढ़ना।

उदाहरण के लिए, C में `/dev/urandom` का उपयोग करना:

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    FILE *fp;
    unsigned int randomNumber;

    // /dev/urandom को पढ़ने के लिए खोलें
    fp = fopen("/dev/urandom", "r");

    // एक यादृच्छिक संख्या पढ़ें
    fread(&randomNumber, sizeof(randomNumber), 1, fp);

    // यादृच्छिक संख्या प्रिंत करें
    printf("Random Number: %u\n", randomNumber);

    // फाइल बंद करें
    fclose(fp);

    return 0;
}
```

यह विधि सीधे सिस्टम के एंट्रोपी पूल से पढ़ती है, जो अधिक संवेदनशील अनुप्रयोगों के लिए उपयुक्त उच्च गुणवत्ता वाली यादृच्छिकता प्रदान करती है। हालांकि, इस दृष्टिकोण में विभिन्न प्लेटफार्मों में पोर्टेबिलिटी के मुद्दे हो सकते हैं, जिससे यह `rand()` का उपयोग करने से कम सार्वभौमिक होता है।

विधि के बावजूद, C में यादृच्छिकता की प्रकृति और इसके कार्यान्वयन को समझना प्रभावी, सुरक्षित, और आकर्षक अनुप्रयोगों को विकसित करने के लिए महत्वपूर्ण है।
