## क्या और क्यों?
कोड को फ़ंक्शन्स में व्यवस्थित करना इसे पुन: उपयोग करने योग्य भागों में तोड़ने के बारे में है। इससे आपका कोड साफ, पढ़ने में आसान, और डीबग करने में सरल हो जाता है।

## कैसे:
यहाँ एक Go स्निपेट है जो एक कोड ब्लॉक को दिखाता है, इसके बाद फ़ंक्शन्स का उपयोग करते हुए एक पुनर्गठित संस्करण है:

```go
package main

import "fmt"

func main() {
    // इससे पहले: सीधे कोड
    fmt.Println("योग की गणना कर रहा है...")
    total := 0
    for i := 1; i <= 10; i++ {
        total += i
    }
    fmt.Println("कुल योग है:", total)

    // बाद में: एक फ़ंक्शन का उपयोग करके
    fmt.Println("फ़ंक्शन का उपयोग करके योग की गणना कर रहा है...")
    sum := getSum(1, 10)
    fmt.Println("कुल योग है:", sum)
}

// एक रेंज के भीतर योग की गणना करने के लिए फ़ंक्शन
func getSum(start, end int) int {
    total := 0
    for i := start; i <= end; i++ {
        total += i
    }
    return total
}
```

सीधे और फ़ंक्शन-आधारित कोड के लिए नमूना आउटपुट समान होगा:

```
योग की गणना कर रहा है...
कुल योग है: 55
फ़ंक्शन का उपयोग करके योग की गणना कर रहा है...
कुल योग है: 55
```

## गहराई से डाइव
फ़ंक्शन्स की अवधारणा उभरने से पहले, प्रोग्रामिंग बड़े पैमाने पर प्रक्रियात्मक थी, कोड शीर्ष से निचे चलता था। जैसे-जैसे प्रोग्राम बड़े होते गए, इस दृष्टिकोण ने अक्षमता और कोड पुनरावृत्ति को उत्तेजित किया।

भाषाओं ने एक सार तंत्र के रूप में फ़ंक्शन्स का परिचय दिया। Go में, फ़ंक्शन्स एक विशेष कार्य के साथ कोड के ब्लॉक्स को समाहित करते हैं, DRY (डोन्ट रिपीट योरसेल्फ) सिद्धांत को प्रोत्साहित करते हैं। वे मानदंडों को स्वीकार करते हैं और परिणाम लौटा सकते हैं।

उपयोगी सुझाव:
- फ़ंक्शन्स का नाम स्पष्ट रखें; एक अच्छा नाम बताता है कि फ़ंक्शन क्या करता है।
- उन्हें छोटा रखें; अगर एक फ़ंक्शन बहुत कुछ करता है, तो इसे तोड़ दें।
- फ़ंक्शन्स कई मान लौटा सकते हैं, एरर हैंडलिंग के लिए इसका लाभ उठाएं।
- उच्च-क्रम फ़ंक्शन्स (फ़ंक्शन्स जो अन्य फ़ंक्शन्स लेते हैं या लौटाते हैं) Go में शक्तिशाली उपकरण हैं।

फ़ंक्शन्स के विकल्प में सीधे कोड (जटिल कार्यों के लिए गन्दा) और ऑब्जेक्ट मेथड्स (स्ट्रक्ट्स के माध्यम से Go में उपलब्ध ऑब्जेक्ट-ओरिएंटेड पैराडाइम) शामिल हैं।
