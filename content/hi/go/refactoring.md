## क्या और क्यों?
रिफैक्टरिंग बाहरी व्यवहार को बदले बिना मौजूदा कम्प्यूटर कोड की पुनर्रचना की प्रक्रिया है। कार्यक्रमकर्ता इसे सॉफ़्टवेयर के गैर-क्रियात्मक गुणों जैसे पठनीयता और रख-रखावशीलता को बेहतर बनाने के लिए करते हैं, जो कोड को आसानी से समझने, जटिलता को कम करने, और बग्स को अधिक आसानी से पहचानने में सहायता कर सकता है।

## कैसे करें:
आइए एक साधारण Go कोड रिफैक्टरिंग उदाहरण में गोता लगाएँ। हम एक स्निपेट लेंगे जो संख्याओं के एक स्लाइस का औसत निकालता है और इसे स्पष्टता और पुन: उपयोगशीलता के लिए रिफैक्टर करेंगे।

मूल कोड:
```Go
package main

import "fmt"

func main() {
    numbers := []float64{8, 12, 15, 10, 7, 14}
    var sum float64
    for _, num := range numbers {
        sum += num
    }
    average := sum / float64(len(numbers))
    fmt.Println("Average:", average)
}
```

रिफैक्टर किया गया कोड:
```Go
package main

import "fmt"

// CalculateAverage एक स्लाइस ऑफ फ्लोट64 लेता है और औसत लौटाता है।
func CalculateAverage(numbers []float64) float64 {
    sum := 0.0
    for _, num := range numbers {
        sum += num
    }
    return sum / float64(len(numbers))
}

func main() {
    numbers := []float64{8, 12, 15, 10, 7, 14}
    average := CalculateAverage(numbers)
    fmt.Println("Average:", average)
}
```

रिफैक्टर किए गए कोड में, हमने औसत की गणना करने वाले लॉजिक को एक अलग फ़ंक्शन `CalculateAverage` नाम से निकाला है। इससे `main` फ़ंक्शन अधिक संक्षिप्त हो जाता है और औसत गणना लॉजिक पुन: उपयोगशील और परीक्षण योग्य बन जाता है।

## गहराई में
कोड रिफैक्टरिंग कोई आधुनिक संकल्पना नहीं है; यह व्यापक कम्प्यूटर उपयोग से पहले का है। यह प्रथा संभवत: मैकेनिकल इंजीनियरिंग के क्षेत्र में या उससे भी पहले शुरू हुई थी। सॉफ़्टवेयर में, यह 1990 के दशक में ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग और एक्सट्रीम प्रोग्रामिंग (XP) के आगमन के साथ अधिक औपचारिक रूप से हुआ, विशेष रूप से मार्टिन फौलर की प्रमुख पुस्तक "रिफैक्टरिंग: एग्ज़िस्टिंग कोड के डिज़ाइन को सुधारना" से प्रभावित हुआ।

रिफैक्टरिंग तकनीकें अनेक हैं, स्पष्टता के लिए वेरिएबल्स का सरल नामकरण से लेकर अधिक जटिल पैटर्न्स जैसे कि मेथड्स या क्लासेस को निकालना तक होती हैं। मुख्य उद्देश्य छोटे, क्रमिक परिवर्तन करना होता है जो सॉफ़्टवेयर की कार्यक्षमता को नहीं बदलते लेकिन आंतर�क संरचना में सुधार करते हैं।

Go का उपयोग करते समय, सरल और शक्तिशाली मानक पुस्तकालय के कारण रिफैक्टरिंग सीधी हो सकती है। हालां�, रिफैक्टरिंग के कारण बग्स का परिचय न हो, इसे सुनिश्चित करने के लिए एक अच्छा सेट ऑफ यूनिट टेस्ट्स होना महत्वपूर्ण है। `gorename` और `gofmt` जैसे टूल्स कुछ प्रक्रियाओं को स्वचालित करने में मदद करते हैं, और IDEs अक्सर बिल्ट-इन रिफैक्टरिंग सपोर्ट प्रदान करते हैं।

मैन्युअल रिफैक्टरिंग के अलावा, Go के लिए कुछ स्वचालित कोड रिफैक्टरिंग टूल उपलब्ध हैं, जैसे GoLand की रिफैक्टरिंग टूल्स और Go Refactor। जबकि ये प्रक्रिया को तेज़ कर सकते हैं, वे कोड को समझने और विचारशील परिवर्तन करने के लिए एक प्रतिस्थापन नहीं हैं।
