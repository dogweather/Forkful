## क्यों

क्या आपने कभी सोचा है कि अपने प्रोग्रामिंग में बग्स को कैसे ठीक किया जाए? आमतौर से हम लॉग फ़ाइल में लोग करते हैं लेकिन क्या आपको पता है कि आप कैसे standard error में भी लोग कर सकते हैं? इस ब्लॉग पोस्ट में हम आपको बताएंगे कि आप क्यों standard error में लोग करना चाहिए।

## कैसे करें?

यदि आप Go प्रोग्रामिंग में नए हैं, तो शायद आपने सुना होगा कि लॉगिंग के लिए ```fmt.Println()``` का उपयोग करना चाहिए। लेकिन आपको यह नहीं पता होगा कि आप standard error में लोग करने के लिए कैसे इस्तेमाल कर सकते हैं। निम्न उदाहरण में, हम एक सामान्य गलती को ठीक करने के लिए standard error में लोग करने के एक उचित तरीके को दिखाएँगे:

```Go
package main

import (
	"fmt"
	"os"
)

func main() {
	// गलती को पकड़ें
	_, err := os.Open("non_existent_file.txt")
	if err != nil {
		// standard error में लोग करें
		fmt.Fprintln(os.Stderr, "गलती:", err)
	}
}
```

आप देख सकते हैं कि हमने अपने कोड के अंत में ```fmt.Fprintln()``` का उपयोग किया है जो कि standard error में लोग करता है। इसके अलावा, हम अपने कोड को भी दो अलग गर्त्तों में बांटते हैं - एक जहाँ सामान्य ```fmt.Println()``` लॉग करता है और दूसरा जहाँ हम standard error में लोग करते हैं। यह उचित अलगाव हमेशा लगाएं ताकि हम अपने लॉग को अलग-अलग गंतव्यों में देख सकें। उदाहरण के लिए, आप इसका उपयोग करके अपने कोड को डेबग करने के लिए इस्तेमाल कर सकते ह
