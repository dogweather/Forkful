## क्यों
हम सभी जानते हैं कि कोडिंग में गलतियों से बचना बहुत महत्वपूर्ण है। लेकिन हमारे क्या हमें यकीन है कि हमारा कोड जो कुशलतापूर्वक कंपाइल होता है, वह वास्तव में सही काम करेगा? इस सवाल का जवाब है आपके टेस्ट। टेस्टिंग हमारे कोड की गुणवत्ता का परख होता है और हमें यकीन दिलाता है कि हमारा कोड सही है।

## कैसे करें
यदि आप रस्ट प्रोग्रामिंग में टेस्टिंग सीखना चाहते हैं, तो आपको चिंता करने की जरूरत नहीं है। रस्ट एक सुरक्षित और प्रफेशनल प्रोग्रामिंग भाषा है जो टेस्टिंग के लिए विशेष फीचर्स प्रदान करती है। आप निम्नलिखित तरीके को फॉलो करके आसानी से अपने कोड को टेस्ट कर सकते हैं।

```rust
fn add(x: i32, y: i32) -> i32 {
   x + y
}

#[test]
fn test_add() {                // यह टेस्ट फंक्शन की पहचान है
   assert_eq!(add(2,3), 5);    // यहां हम यह देख रहे हैं कि add फंक्शन 2 और 3 को जोड़कर 5 दे रहा है या नहीं।
}

```

इस उदाहरण में, हमने दो संख्याओं को जोड़ने के लिए एक फंक्शन बनाया है और उसे टेस्ट किया है कि क्या वह सही रिजल्ट दे रहा है। यदि कोई गलती होती है, तो टेस्ट केस फेल हो जाएगा और हम उसे ठीक कर सकते हैं।
