## क्या और क्यों?
कोड को फंक्शन्स में आयोजित करने के पीछे का विचार यह है कि स्क्रिप्ट की छोटी छोटी हिस्से एकत्रित करके विशिष्ट कार्य करना है। हम इसे इसलिए करते हैं क्योंकि यह कोड को पढ़ने, परीक्षण करने और पुनः प्रयोग करने के लिए आसान बनाता है — किसी को भी कोड की स्पघेटी के झंझावात में फंसना पसंद नहीं है।

## कैसे करें:
फिश में, आप `function` कीवर्ड का उपयोग करके, इसे एक नाम दें, और `end` से समाप्त करें। यहाँ एक सरल उदाहरण है:

```fish
function hello
    echo "Hello, World!"
end

hello
```

आउटपुट:
```
Hello, World!
```

अब, चलिए इसे किसी यूजर को अभिवादन करने के लिए बनाते हैं:

```fish
function greet
    set user (whoami)
    echo "Hey there, $user!"
end

greet
```

आउटपुट:
```
Hey there, your_username!
```

इसे सत्रों में सहेजने के लिए, `funcsave greet` का उपयोग करें।

## गहराई से जानकारी
फिश शेल फंक्शन्स मिनी-स्क्रिप्ट्स की तरह होते हैं — आप इसमें करीब-करीब कुछ भी डाल सकते हैं। ऐतिहासिक रूप से, शेल स्क्रिप्टिंग में फंक्शन्स की अवधारणा ने दोहराए जाने वाले टाइपिंग और डिबगिंग के अनगिनत घंटे बचाए हैं। पाइथन जैसी प्रोग्रामिंग भाषाओं की तुलना में, शेल फंक्शन्स ढांचे की बजाय सुविधा के बारे में अधिक होते हैं।

कुछ शेल्स, जैसे कि बैश, `function` या सीधे कोष्ठकों का उपयोग करते हैं। फिश `function ... end` का उपयोग करते हैं — स्पष्ट और पढ़ने में आसान। फिश फंक्शन्स के अंदर, आपको सभी सुविधाएँ मिलती हैं: मापदंड, स्थानीय चर परिभाषित करने के लिए `set -l`, और आप यहां तक की एक फंक्शन के अंदर एक और फंक्शन भी परिभाषित कर सकते हैं।

आपको `return` मान की आवश्यकता नहीं है क्योंकि फिश इस पर बड़ा ध्यान नहीं देता; आपके फंक्शन का आउटपुट ही इसकी रिटर्न होती है। और अगर आप चाहते हैं कि फंक्शन्स भविष्य के सत्रों के लिए स्थायी रूप से उपलब्ध हों, तो `funcsave` का उपयोग करें।
