## क्यों
एक नया प्रोजेक्ट शुरू करने में एंगेज होने का *क्यों* कारण सिर्फ 1-2 सेंटेंस में बताएं।

एक नया प्रोजेक्ट शुरू करना एक रोमांचक कार्य हो सकता है। इससे आपको नयी चुनौतियों का सामना करने का मौका मिलता है और अपनी कौशल को नए स्तर पर ले जाने की संभावना होती है।

## कैसे
कोडिंग उदाहरण और "```Haskell ... ```" कोड ब्लॉक के भीतर सैंपल आउटपुट।

**उदाहरण 1: डाटा टाइप्स बनाना**

```Haskell
data Book = Book { title :: String, author :: String, year :: Int }

let myBook = Book { title = "हैस्केल प्रोग्रामिंग", author = "जॉन स्मिथ", year = 2020 }

print(title myBook) -- इस्ट्रेंग आउपुत: "हैस्केल प्रोग्रामिंग"
```

**उदाहरण 2: रिकर्सन फंक्शन**

```Haskell
factorial :: Int -> Int
factorial 0 = 1
factorial n = n * factorial (n-1)

print(factorial 5) -- आउपुट: 120
```

## गहराई संविधान
नए प्रोजेक्ट शुरू करने के बारे में अधिक जानकारी। इसमें निम्नलिखित सामग्री शामिल है:

- प्रोजेक्ट की आवश्यकता और उसके उपयोग का विश्लेषण
- प्रोजेक्ट में शामिल होने वाली भागों और फंक्शन के स्रोत को आसानी से समझने के लिए अंकित करने का तरीका
- अलग-अलग कोडिंग प्रैक्टिस का उपयोग करके प्रोजेक्ट को आगे बढ़ाने के लिए सुझाव
