# "Gleam में डायरेक्टरी मौजूद है या नहीं, कैसे जांचें?" 

## "क्या & क्यों?" (What & Why?)
डायरेक्टरी मौजूद है या नहीं, जांचना एक तरह से हमारे सिस्टम में एक विशेष पथ पर एक फ़ोल्डर की उपस्थिति की पहचान है। प्रोग्रामर्स इसे करते हैं ताकी त्रुटियां से बच सकें, जैसे कि फ़ाइल को एक गैर-मौजूद डायरेक्टरी में लिखना।

## "कैसे?" (How to?)
Gleam लैंग्वेज में, हम [`gleam/erlang`] लाइब्रेरी का उपयोग करके यह पता लगा सकते हैं कि क्या एक डायरेक्टरी मौजूद है या नहीं।

```gleam
आयात gleam/erlang

फ़ंक्शन dir_exists(dir: तार) -> बूल {
  erlang.fs_mkdir(dir)
  |> खास.पैटर्न मिलावें
  ok -> नकली
  { error, eexist } -> सही
  _ -> नकली
}
```
ध्यान दें कि यह केवल तब काम करेगा जब आपके पास डायरेक्टरी का पूर्ण पथ हो।

## "गहरी डाइव" (Deep Dive)
विगत के समय में, डायरेक्टरी की जांच के लिए विभिन्न तकनीकों का उपयोग हुआ है, जैसे लगातार प्रयास करना जब तक कि त्रुटि संदेश मिल न जाए। हालांकि, `gleam/erlang.fs_mkdir` और पैटर्न मैचिंग का उपयोग करने के द्वारा, हम आसानी से त्रुटियों से बच सकते हैं। यह तकनीक अधिक सुरक्षित और विश्वसनीय है, चूंकि हम सीधे ऑपरेटिंग सिस्टम के कर्नल से जानकारी प्राप्त करते हैं।
