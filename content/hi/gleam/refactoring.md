## क्या और क्यों?
रिफैक्टरिंग एक प्रक्रिया है जिसमें आप अपने कोड को पुनर्गठित करते हैं ताकि वह स्वच्छ, अधिक रखरखाव योग्य बने, बिना इसके बाहरी व्यवहार में परिवर्तन किए। प्रोग्रामर रिफैक्टरिंग करते हैं ताकि पठनीयता में सुधार हो, जटिलता कम हो, और कोडबेस को भविष्य के अपडेट्स या फीचर जोड़ने के लिए अधिक अनुकूल बनाया जा सके।

## कैसे:
मान लीजिए आपके पास कोड का एक टुकड़ा है जहां आप कई समारोहों में कुछ पुनरावृत्ति गणनाएँ या स्ट्रिंग मनिपुलेशन कर रहे हैं। यह रिफैक्टरिंग के लिए एक प्रमुख लक्ष्य है। यहाँ ग्लीम का उपयोग करते हुए पहले और बाद का एक उदाहरण है, जिसमें टाइप सुरक्षा और अपरिवर्तनीयता पर जोर दिया गया है:

```gleam
// रिफैक्टरिंग से पहले
pub fn calculate_area(width: Int, height: Int) -> Int {
  width * height
}

pub fn print_area(width: Int, height: Int) {
  let area = calculate_area(width, height)
  io.println("क्षेत्रफल है \(area)")
}

// रिफैक्टरिंग के बाद
pub fn calculate_area(width: Int, height: Int) -> Int {
  width * height
}

pub fn print_area(area: Int) {
  io.println("क्षेत्रफल है \(area)")
}

// आपके कोड के दूसरे हिस्से में, आप इस तरह से print_area को कॉल करेंगे:
print_area(calculate_area(10, 20))
```

नमूना आउटपुट:
```
क्षेत्रफल है 200
```

रिफैक्टरिंग द्वारा, हमने `print_area` को केवल प्रिंटिंग पर अधिक केंद्रित बनाया है, जबकि गणना को अन्य जगह पर संभाला जाता है, जिससे कोड अधिक मोड्यूलर और पुन: उपयोग या परीक्षण के लिए आसान बनता है।

## गहराई से अध्ययन
रिफैक्टरिंग, एक अवधारणा के रूप में, प्रोग्रामिंग के समान लंबे समय से है—कोड को पुनः देखना और साफ-सफाई अच्छे गृह-कार्य का एक हिस्सा है। रिफैक्टरिंग का आधुनिक औपचारिकीकरण, साथ ही आज उपयोग की जाने वाली कई तकनीकें और पैटर्न, 1999 में प्रकाशित मार्टिन फाउलर की प्रमुख पुस्तक "Refactoring: Improving the Design of Existing Code" के लिए वापस ट्रेस की जा सकती हैं।

ग्लीम इकोसिस्टम में, रिफैक्टरिंग के विशिष्ट विचार हैं। सबसे महत्वपूर्ण में से एक संकलन समय पर मजबूत टाइप जाँच है, जो आपको चीजों को इधर-उधर करते समय जल्दी से गलतियों को पकड़ने में मदद कर सकता है। ग्लीम की पैटर्न मैचिंग और अपरिवर्तनीयता की विशेषताएं भी आपको स्पष्ट, अधिक संक्षिप्त कोड लिखने के लिए मार्गदर्शन कर सकती हैं—जो रिफैक्टरिंग के प्रमुख लक्ष्यों में से एक है।

रिफैक्टरिंग के विकल्पों में शामिल हो सकते हैं कोड को खरोंच से फिर से लिखना या कोड को जल्दी से ठीक करना। हालांकि, रिफैक्टरिंग आमतौर पर मौजूदा कोड को सुधारने का सबसे सुरक्षित और सबसे कुशल तरीका है जो नए बग्स को पेश किए बिना, क्योंकि यह क्रमिक, अच्छी तरह से अंतर्निहित, व्यवहार-संरक्षण परिवर्तनों में शामिल होता है।
