## מה ולמה?
הדפסת אוטפוט לניפוי באגים היא כלי לצורך בדיקת ועקיבה אחרי התנהלות התוכנה בזמן ריצה. תכנתים משתמשים בהדפסות לזיהוי בעיות ולהבנת תהליכים בקוד.

## איך לעשות:
```gleam
fn main() {
  let some_value = "הערך שלי";
  dbg(some_value); // הדפסת הערך לקונסול לניפוי באגים
}

// פלט דוגמה:
// > "הערך שלי"
```
אמנם Gleam טרייה יחסית, אך השימוש ב`dbg` פשוט וישיר. הוסיפו `dbg(ערך);` והערך יודפס לקונסול.

## צלילה עמוקה
בעבר, למפתחים היו פחות אפשרויות ניפוי באגים והם השתמשו בהדפסת טקסט רגילה, כמו `println!` ב-Rust או `printf` ב-C. היום, כלים כמו `dbg` ב-Gleam מספקים מידע נוסף כמו מיקום בקוד של ההדפסה. בנוסף לכך, יש אלטרנטיבות כמו מנגנוני לוגינג שונים או כלים גרפיים לניפוי באגים אך להדפסה יש יתרונות בפשטות ובזמינות.
