## מה ולמה?
עיגול מספרים הוא התאמת ערך למקום המבוקש הקרוב ביותר—כמו מ-2.56 ל-3 אם אנו מעגלים למספרים שלמים. מתכנתים עושים זאת מטעמי פשטות או כדי לעמוד במפרטים מספריים מסוימים, בדרך כלל כדי להימנע מעדינויות הנגרמות על ידי שגיאות דיוק שברים צפים או כדי להפוך את הפלט לידידותי למשתמש.

## איך לעשות:
ב-Gleam, עיגול אינו נמצא בספריית הסטנדרט נכון לבדיקה האחרונה שלי, אך הנה איך בדרך כלל תעגלו מספר צף למספר שלם הקרוב באמצעות פונקציות של Erlang ישירות:

```gleam
external fn erlang_round(Float) -> Int = "erlang" "round"

pub fn main() {
  let rounded = erlang_round(2.56)
  rounded // פלט: 3
}
```

פלט:
```
3
```

יש לכם דיוק שונה בראש? למשל, עיגול לשני מקומות עשרוניים? נצטרך קצת מתמטיקה:

```gleam
pub fn round_to_two_places(num: Float) -> Float {
  let multiplier = 100.0
  let tmp = num * multiplier
  let round_tmp = erlang_round(tmp)
  round_tmp / multiplier
}

pub fn main() {
    round_to_two_places(2.569) // פלט: 2.57
}
```

פלט:
```
2.57
```

## צלילה לעומק
בהיסטוריה, עיגול מספרים היה קריטי, במיוחד בחישובים כלכליים ומדעיים שבהם דיוק ותקנים חשובים מאוד. ללא עיגול, הייתם מקבלים עשרוניים ארוכים ומכוערים בכל מקום, מה שהיה הופך את החישובים לבלתי פרקטיים ונוטים לשגיאות.

בעולם התכנות, שפות שונות מציעות גישות שונות, מפונקציות מובנות ועד ספריות מתמטיות מקיפות. עיגול עשוי לכלול חוקים שונים - למשל, "עגל חצי למעלה" (השיטה הרגילה) או "עגל חצי לזוגי" (שגרתית נעשית בחישובים כלכליים כדי להימנע מהטייה).

Gleam, שפה צעירה עם שורשים ב-Erlang, מסתמכת על סט מתמטי נרחב של פונקציות מספריות של Erlang. ככל שהשפה תגדל, ייתכן שנראה פונקציות פנימיות חדשות, מה שיקטין את הצורך לקרוא לשגרות חיצוניות.
