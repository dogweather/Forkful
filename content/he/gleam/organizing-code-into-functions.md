## מה ולמה?
ארגון קוד לתוך פונקציות פירושו שבירת התנהגות של תוכנית לתתי חלקים קטנים וניתנים לשימוש חוזר. מתכנתים עושים זאת כדי להפוך את הקוד לברור יותר, לקל על תחזוק שלו וכדי להימנע מחזרה.

## איך לעשות:
הנה דוגמא פשוטה לאירגון קוד לתוך פונקציות ב-Gleam:

```gleam
fn add(x, y) {
  x + y
}

fn main() {
  let sum = add(3, 4)
  sum
}

// פלט לדוגמא
// 7
```

בקטע זה, `add` היא פונקציה שלוקחת שני ערכים ומחברת אותם. `main` היא המקום שבו אנו קוראים ל `add` ומנהלים את התוצאה.

## עיון עמוק
היסטורית, המושג של פונקציות (או 'תת-תכניות') השפיע עמוקות על תכנות, ופתח את הדרך לתכנות מבני בשנות ה-60 ואילך. פונקציות מעודדות גישה מודולרית, בה בעיות מחולקות לתת-בעיות, נפתרות באופן עצמאי, ומורכבות כדי לפתור את הבעיה הגדולה יותר.

ב-Gleam, שהוא מסוגן בצורה חזקה, פונקציות כוללות גם מידע על סוגים, אשר מבטיח שהשימוש בהן יתאים להגדרה שלהן. דבר זה מפחית טעויות ומבהיר כוונות.

אלטרנטיבות לפונקציות כוללות קידוד פנימי, בו הלוגיקה כתובה שוב ושוב. למרות שלעיתים זה מהיר יותר למשימות קטנות וחד-פעמיות, קידוד פנימי אינו מתרחב היטב עבור יישומים גדולים יותר.

פרטים ליישום שכדאי לשקול כשמארגנים לתוך פונקציות יכולים לכלול הרכבת פונקציות, בה פונקציות משמשות כבלוקים בנייה, ופונקציות מסדר גבוה יותר, שלוקחות פונקציות אחרות כארגומנטים או שמחזירות אותם, מוסיפות גמישות לכיצד הקוד מאורגן ומבוצע.
