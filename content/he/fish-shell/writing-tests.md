## למה

כתיבת טסטים היא חלק חשוב בעבודת התכנות המקצועית שלך. היא מבטיחה שהקוד שלך עובד כפי שאתה מצפה ומונעת באגים פוטנציאליים. תוכל לחסוך זמן רב בעתיד על באגים שכבר נגזרו על ידי הוכחות בטסטים.

## איך לעשות זאת

תחילה, נצור תיקייה חדשה לפרויקט התכנות שלנו ונכנס אליה. לעבוד בתיקייה זו, יתכן שתצטרך להשתמש בפקודה '`mkdir`' כדי ליצור אותה. בתיקייה זו, נייצר קובץ חדש עם השם 'tests.fish'.

```Fish Shell
$ mkdir my_project
$ cd my_project
$ touch tests.fish
```

עכשיו נוכל להתחיל לכתוב את הטסטים שלנו בתוך הקובץ החדש הזה. נוכל להשתמש בסינטקס הבא כדי להוסיף טסטים לקובץ:

```Fish Shell
@ my_first_test
# here we write the code for our first test
```

בכל פעם שנרצה להריץ את הטסטים שכתבנו, ניתן פשוט להריץ את הקובץ 'tests.fish'.

```Fish Shell
$ ./tests.fish
```

בהנחה שלא היו שגיאות, תתבצע כל הקודים שכתבנו ונקבל כתוצאה את הודעת ההצלחה הבאה:

```
All tests passed!
```
