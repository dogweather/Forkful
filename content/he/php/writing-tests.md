# למה

כתיבת בדיקות בתוכנות היא נושא חשוב לכל מפתחי התוכנה. בעזרת בדיקות אנחנו יכולים לוודא שקודנו תקין ובטח, מה שמבטיח עבודת עדרת רחפן של הפעלות ללא בעיות כאשר התוכנית מורידה באצרך לאו הינו מי פאסט. 

כמו כן, בדיקות יכולות לחסוך לנו את זמן ומאמץ רב באיתור ותיקוני תקלות כשאנחנו בונים את התוכנה הסופר בשלום. 

# איך לעשות

באמצעות  <code> ```PHP ... ``` </code> נוכל לדלג על פאסט קודנו בתוכן הטקסט ולהציגו בפורמט של קוד פתוח במהירות ובנוחות. 

לדוגמה, ננסה לכתוב בדיקת יחידות פשוטה עבור פונקציה שמקבלת שתי מחרוזות ומחזירה את רווח בניהם:

```PHP
function concatStrings($str1, $str2) {
  return $str1 . " " . $str2;
}

$concatResult = concatStrings("שלום", "עולם");

echo $concatResult;

// Output: שלום עולם
```

במקרה זה, אנחנו בודקים שהפונקציה מחזירה את התוצאה הנכונה על פי המחרוזות המצורפות. כך, אם נערוך שינויים בקוד, נוכל להשתמש בבדיקות יחידות כדי לוודא שהם לא השפיעו על התוכנית המתפעלת בלעדנו ידע מערכתי על בעיות שנוצרו. 

# מגה מצא

כמו שאפשר לראות, כתיבת בדיקות יחידות היא כלי חשוב בתהליך הפיתוח שלנו. כדי ללמוד עוד על נושא זה, ניתן לעיין במאמרים נוספים המפרטים יתרונותיו של הבדיקות ויחידות ואיך ליישם אותם בתוכנות שלנו.
