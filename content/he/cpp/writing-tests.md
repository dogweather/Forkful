##למה

כתיבת בדיקות היא חלק חשוב מתהליך התכנות שכולנו אוהבים וחושפת לנו כמה נקודות מאוד חשובות של הקוד שלנו. בדיקות מאפשרות לנו לוודא שהתוכנית שכתבנו עובדת כפי שהתכנן ואין בעיות כלשהן שיכולות לגרום לקוד להתקלקל. בנוסף, זה יכול לחסוך לנו זמן רב בעתיד כאשר אנחנו מוסיפים יישומים נוספים או מבצעים שינויים.

##כיצד

```C++
//לדוגמה:
#include <iostream>
#include <cassert>

int main ()
{
  int x = 5;
  std::cout<<"x = " << x << std::endl;
  
  //עכשיו בודקים את התוצאה הצפויה
  assert(x == 5);
  
  return 0;
}
```

תוכניות עם בדיקות צריכות להיות כתובות כך שהבדיקה צועדת על הקוד החשוב ביותר בתוכנית או בתוך פונקציית עיקוב עם בדיקות שונות לכל קלט אפשרי. השתמש בassert כדי לוודא שהתוצאות שציפית להם הן מדוייקות.

##עיון מעמיק

כאשר אנו מכתיבים בדיקות, חשוב לשים לב לפרטים קטנים כגון בדיקת כמה קלטים שונים יש לנו ואילו פונקציות צריכות לקבל פרמטרים אפשריים. כמו כן, חשוב לוודא שהאלגוריתמים שכתבנו עובדים נכון גם עבור קלטים בגרות או שגיאות.
