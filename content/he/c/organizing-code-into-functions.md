## מה ולמה?
ארגון קוד לתוך פונקציות זו תהליך שבו מחלקים את הקוד לבלוקים ניתנים לשימוש חוזר אשר מבצעים משימות ספציפיות. זה מקל על קריאת הקוד, ניפוי שגיאות, ותחזוקה.

## איך לעשות:
בואו ניקח דוגמה פשוטה: נניח שאתם רוצים לחבר שני מספרים פעמים רבות.

בלי פונקציות:
```C
#include <stdio.h>

int main() {
    int sum1 = 5 + 3;
    printf("Sum1: %d\n", sum1);
    
    int sum2 = 2 + 8;
    printf("Sum2: %d\n", sum2);
    
    // עוד חיבורים כאן...
    
    return 0;
}
```

עם פונקציות:
```C
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int sum1 = add(5, 3);
    printf("Sum1: %d\n", sum1);
    
    int sum2 = add(2, 8);
    printf("Sum2: %d\n", sum2);
    
    // השתמשו בפונקציית add() לעוד חיבורים...
    
    return 0;
}
```

פלט:
```
Sum1: 8
Sum2: 10
```

## ניתוח עמוק
לפני שהיו פונקציות בשפת C, תכנות היה לעתים קרובות לינארי, כמו מתכון. אך כאשר התוכניות התרחבו, כפילות קוד הפכה לבעיה. פונקציות היו הפתרון - הן אפשרו לנו לבצע את אותו בלוק קוד ממקומות שונים בתכנית מבלי לכתוב אותו מחדש בכל פעם. זה לא רק חוסך מקום אלא גם זמן כאשר עושים עדכונים: שנו את הפונקציה במקום אחד, וכל חלק בקוד שלכם שמשתמש בה מקבל את העדכון.

אלטרנטיבות לפונקציות עשויות לכלול קוד אינליין, מאקרואים, או קידוד בעזרת העתק-הדבק, אבל אלו עלולים להוביל לקוד נפוח, רגיש לשגיאות וקשה לתחזוקה. פונקציות, לעומת זאת, מקפסלות פונקציונליות, מגדירות ממשקים ברורים, ויכולות להפחית תופעות לוואי באמצעות שימוש נכון בסקופ (היקף).

כאשר אתם מיישמים פונקציות, שקלו כמה פרטים: ראשית, נסו להפוך אותן לבצע רק דבר אחד – זה מוכר בשם עקרון האחריות הבודדת. שנית, שמות הם חשובים – בחרו שמות תיאוריים לפונקציות ולפרמטרים שלהן למינוף הקוד שלכם למסמכים עצמיים.
