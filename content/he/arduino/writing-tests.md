כניסה לעולם הארדואינו: למה לבדוק?

בעולם התכנות הסבוך של הארדואינו, נראה כי אין קצה לסופכם של האתגרים. אחת הדרכים הטובות ביותר בלתי מכוננת ללמוד כיצד להתמודד עם אתגרים כזה היא להתחיל להשתמש במבחנים. במאמר הזה, אנו נוכל לראות למה אנחנו צריכים לבדוק את הקוד שלנו ואיך להתחיל לכתוב בדיקות בארדואינו.

## למה לבדוק?

בדיקות הן חשובות מאוד לכל תהליך תכנות, במיוחד כאשר מדובר בארדואינו. הן מאפשרות לנו לבדוק את הקוד שלנו ולזהות בעיות כבר בשלב מוקדם, לפני שנריץ אותו על התוכנית האמיתית. בנוסף, בדיקות עוזרות לנו לצמצם את זמן האימות ולמנוע באופן כללי בעיות טכניות.

## איך לבדוק?

נציג שתי דוגמאות לכתיבת בדיקות בארדואינו, אחת לבדיקת כפתור והשניה לבדיקת אנימציה פשוטה. נתחיל עם הבדיקה הראשונה:

```Arduino
int buttonPin = 2; // פלט לכפתור
int ledPin = 13; // פלט ללד

void setup() {
  pinMode(ledPin, OUTPUT);
}

void loop() {
  int state = digitalRead(buttonPin); // קריאת המצב של הכפתור
  
  if(state == HIGH) { // אם הכפתור נלחץ, הלד יידלק
    digitalWrite(ledPin, HIGH);
  } else { // אם הכפתור לא נלחץ, הלד ייכבה
    digitalWrite(ledPin, LOW);
  }
}
```

ניתן לראות כי אנחנו משתמשים כאן בפינג של הכפתור כדי לקרוא את המצב שלו ולהתאים את הפלט של הלד בהתאם. לאחר שהיסטוריית הפינג של הכפתור והלד הוגדרו בפונקציית האתחול, הפונקציה "loop" מבצעת את הבדיק
