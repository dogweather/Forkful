## מה ולמה?
ארגון קוד לתוך פונקציות מדובר בפיצול הקוד לחתיכות ניתנות לשימוש חוזר. זה גורם לקוד שלך להיות נקי יותר, קל יותר לקריאה ופשוט יותר לאיתור ותיקון שגיאות.

## איך לעשות:
הנה קטע בשפת Go שמראה בלוק של קוד, אחריו גרסה מחודשת בשימוש בפונקציות:

```go
package main

import "fmt"

func main() {
    // לפני: קוד מוטבע
    fmt.Println("מחשב סכום...")
    total := 0
    for i := 1; i <= 10; i++ {
        total += i
    }
    fmt.Println("הסכום הכולל הוא:", total)

    // אחרי: שימוש בפונקציה
    fmt.Println("מחשב סכום באמצעות פונקציה...")
    sum := getSum(1, 10)
    fmt.Println("הסכום הכולל הוא:", sum)
}

// פונקציה לחישוב סכום בתחום מסוים
func getSum(start, end int) int {
    total := 0
    for i := start; i <= end; i++ {
        total += i
    }
    return total
}
```

הפלט לשני הבלוקים, הקוד המוטבע והקוד המבוסס על פונקציה, יהיה זהה:

```
מחשב סכום...
הסכום הכולל הוא: 55
מחשב סכום באמצעות פונקציה...
הסכום הכולל הוא: 55
```

## ניתוח עמוק
לפני שפות התכנות הכירו את מושג הפונקציות, התכנות היה בעיקר פרוצדורלי, עם קוד שרץ מלמעלה למטה. ככל שהתוכניות גדלו, הגישה הזו גרמה לחוסר יעילות וחזרה על הקוד.

שפות הכניסו פונקציות כמנגנון הכללה. בשפת Go, פונקציות מקפסלות בלוקים של קוד עם משימה מסוימת, ומעודדות את עקרון DRY (Don't Repeat Yourself - אל תחזור על עצמך). הן מקבלות פרמטרים ויכולות להחזיר תוצאות.

טיפים שימושיים:
- תנו שמות ברורים לפונקציות; שם טוב מסביר מה הפונקציה עושה.
- שמרו עליהן קצרות; אם פונקציה עושה יותר מדי, פצלו אותה.
- פונקציות יכולות להחזיר מספר ערכים, השתמשו בזה לטיפול בשגיאות.
- פונקציות מסדר גבוה (פונקציות שמקבלות או מחזירות פונקציות אחרות) הן כלים חזקים בGo.

חלופות לפונקציות כוללות קוד מוטבע (בלגני למשימות מורכבות) ושיטות אובייקט (חלק מהפרדיגמה המונחית-עצמים שזמינה בGo באמצעות structs).
