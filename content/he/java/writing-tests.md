למה: עבור מחברים ישנם מספר סיבות לכתוב בדיקות בריאות בקוד שלהם. בין היתר, בדיקות עוזרות לוודא שהקוד עובד כצפוי ולקבל משואות בזמן העריכה במקרים שבהם משהו הולך לא כמו שצריך.

## איך לעשות זאת

בכדי ליצור בדיקות בריאות לקוד שלכם, יש להשתמש באחת מהמסגרות הקיימות בשפת ג'אווה. כדי להתחיל, יש ליצור מחלקת Test ולהעביר לה את הקוד שברצונכם לבדוק. כדי לבדוק את הקוד, יש להשתמש בתשובות assert שקיימות במחלקת Test. להלן דוגמא קוד לבדיקת חיבור של מספרים:

```Java
// הגדרת מחלקת Test
public class MyTest {
  // בדיקת חיוב עבור חיבור של 5 ו- 7
  assert MyMath.add(5, 7) == 12;
  // בדיקת חיוב עבור חיבור של -2 ו- 3
  assert MyMath.add(-2, 3) == 1;
}
```

טכניקה זו תאפשר לכם לבדוק את התוצאות של הקוד שלכם ולקבל משואות במקרים שבהם משהו הולך לא כמו שצריך.  כמו כן, יש לאתחל מחלקת Test בעזרת @Before כדי לוודא שהכל יתבצע בסדר נכון.

## חצי נפילה

ניתן להמר את בדיקות הקוד שלכם לבדיקות עמוקות יותר. לדוגמא, במקום לבדוק רק את התוצאות של כל קטע קוד, תוכלו לוודא שהתוצאות מתאימות לפרמטרים הנכונים ולהשתמש בבדיקות חיקוי אחרות כדי לוודא שהקוד שלכם מכיל את כל התכונות שצריך.
