ניצולים בתכנות כמו בתכנות, יישומי JavaScript נרתחים עם טכנולוגיות חדשניות שמטרתן למזער את כמות הזרימה הנכנסת (input) בפרויקטי תכנות מורכבים. אחת הטכנולוגיות הכי נפוצות היא הבערכת ביטויים רגולריים. בפוסט הזה, אנחנו ננסה להבין למה זה כל כך חשוב להשתמש בהם בקוד TypeScript וכיצד להשתמש בהם.

## למה

ביטויים רגולריים מיועדים לקריאה על גלידרת הנתונים שנמצאת בתוך שורת טקסט ארוכה ומרהיבה כמו קוד TypeScript. זה יכול להיות נוח יותר לכתוב ביטויים רגולריים עבור נתונים מרדפים כדי למצוא במהירות מקומי את הנתונים שאנחנו מחפשים במתוך הטקסט המרתק.

## איך לעשות את זה

מספר קוד דוגמה למציאת רגיל מתפרש למטה:

```TypeScript
const stringToSearch = "אני רק מנסה למצוא חלקים מיוחדים של מלים בתוך הטקסט";
const regex = /אני רק/mg;
console.log(regex.exec(stringToSearch));
```

תוצר: `['אני רק', 'אני רק', index: 0, input: 'אני רק מנסה למצוא חלקים מיוחדים של מלים בתוך הטקסט']`.

מספר קוד דוגמה נוסף ניתן למצוא למטה, הפעם למציאת מספרים ביותר קטנים ברשימה:

```TypeScript
const numbers = [1, 22, 333, 4444, -5555];
const regex = /\D/g;
console.log(numbers.filter(number => number.toString().match(regex)));
```

תוצר: `['1', '22']`.
