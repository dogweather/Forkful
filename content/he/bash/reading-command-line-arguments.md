## מה ולמה?

בניתוח ארגומנטים של שורת הפקודה, אנחנו מאמינים לתוכנית לקרוא ולהפעיל נתונים משורת הפקודה. תכניתיים משתמשים בזה באופן מרובה בשל הגמישות שהתכנית יכולה להציע כאשר משתנים חיצוניים מעורבים.

## איך לעשות?

בהנחה שיש לנו תכנית shell בשם myscript.sh, נוכל לקרוא ארגומנטים משורת הפקודה באופן הבא:

```Bash
#!/bin/bash
echo "הארגומנט הראשון הוא: $1"
echo "הארגומנט השני הוא: $2"
```

הרצת התכנית כאשר מחריזים ארגומנטים כדלקמן:

```Bash
$ ./myscript.sh אהלן שלום
הארגומנט הראשון הוא: אהלן
הארגומנט השני הוא: שלום
```

## צלילה עמוקה

1. תוך יומנמות: שפת Bash נוצרה בשנת 1989 כחלק מפרויקט GNU. במקור נעשה שימוש בעיבוד שורת פקודה בשפת shell, אך עם תפתחו של Bash, מתכנתים התחילו להוסיף ארגומנטים במטרה להגביר את היכולות שלהם.
2. חלופות: ייתכן כי לא כל התסריטים מצריכים משתנים של ארגומנטים. ישנן שיטות חלופות כמו קבצי תצורה, משתני סביבה או קלט מהמשתמש בזמן ריצה.
3. פרטי ביצוע: מדובר במערך שמתחיל מ-0, כאשר $0 מייצג את שם התסריט עצמו. הארגומנטים האחרים ממוספרים בהתאם.
