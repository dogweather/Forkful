## Что и зачем?
Разбор даты означает преобразование строки в объект даты. Программисты делают это для обработки дат в стандартизированном виде для хранения, сортировки или манипулирования.

## Как это сделать:
```Go
package main

import (
	"fmt"
	"time"
)

func main() {
	// Пример строки с датой
	dateStr := "2023-04-01T15:04:05Z"

	// Определение формата, соответствующего указанному выше
	layout := time.RFC3339

	// Разбор строки в объект time.Time
	parsedDate, err := time.Parse(layout, dateStr)
	if err != nil {
		panic(err)
	}
	
	// Вывод
	fmt.Printf("Разобранная дата: %v\n", parsedDate)
}
```
Вывод:
```
Разобранная дата: 2023-04-01 15:04:05 +0000 UTC
```

## Подробнее
Разбор дат всегда был критически важен для программных систем для организации и обработки временной информации с, ну, практически начала вычислений. В Go, `time.Time` это структура, представляющая время. Она создана для простоты и эффективности. Почему использовать строки для начала? Главным образом потому, что даты поступают в виде текста из разных источников, таких как API или пользовательский ввод.

Альтернативы? Ну, теоретически вы могли бы делать ручной разбор, но он подвержен ошибкам. Функция `time.Parse` в Go позволяет вам определить формат – эталонную дату – с которой вы сравниваете ваш ввод. Это надежный метод, потому что начиная с Go 1 (примерно 2012 год), этот психологический критерий для времени, понятного человеку, делает ваш разбор точным. Функциональность, похожая на `datetime` в Python и `SimpleDateFormat` в Java, также доступна, но они не так строги, как реализация разбора в Go, которая не пытается угадать, что вы имели в виду.

Вот что интересно: функция разбора Go требует определенного эталонного времени: `Mon Jan 2 15:04:05 MST 2006` (01/02 03:04:05PM '06 -0700). Запомните эту точную последовательность; многие люди делают это, используя мнемоническую фразу, "1 2 3 4 5 6 7".
