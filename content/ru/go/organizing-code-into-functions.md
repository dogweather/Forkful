## Что и Зачем?
Организация кода в функции заключается в разбиении вашего кода на переиспользуемые части. Это делает ваш код более чистым, легче читаемым и проще в отладке.

## Как это сделать:
Вот фрагмент на Go, который показывает блок кода, за которым следует рефакторинг с использованием функций:

```go
package main

import "fmt"

func main() {
    // До: Встроенный код
    fmt.Println("Расчёт суммы...")
    total := 0
    for i := 1; i <= 10; i++ {
        total += i
    }
    fmt.Println("Общая сумма:", total)

    // После: Использование функции
    fmt.Println("Расчёт суммы с использованием функции...")
    sum := getSum(1, 10)
    fmt.Println("Общая сумма:", sum)
}

// Функция для расчёта суммы в диапазоне
func getSum(start, end int) int {
    total := 0
    for i := start; i <= end; i++ {
        total += i
    }
    return total
}
```

Пример вывода для кода как с встроенным кодом, так и на основе функций, будет одинаковым:

```
Расчёт суммы...
Общая сумма: 55
Расчёт суммы с использованием функции...
Общая сумма: 55
```

## Подробнее
До появления концепции функций программирование было в основном процедурным, с кодом, выполняющимся сверху вниз. По мере роста программ этот подход приводил к неэффективности и повторению кода.

Языки ввели функции как механизм абстракции. В Go функции инкапсулируют блоки кода с конкретной задачей, поощряя принцип DRY (Don't Repeat Yourself - Не повторяйся). Они принимают параметры и могут возвращать результаты.

Полезные советы:
- Именуйте функции чётко; хорошее имя объясняет, что делает функция.
- Делайте их короткими; если функция делает слишком много, разбейте её на части.
- Функции могут возвращать несколько значений, используйте это для обработки ошибок.
- Функции высшего порядка (функции, которые принимают или возвращают другие функции) являются мощным инструментом в Go.

Альтернативы функциям включают встроенный код (запутанный для сложных задач) и методы объектов (часть объектно-ориентированной парадигмы, доступной в Go через структуры).
