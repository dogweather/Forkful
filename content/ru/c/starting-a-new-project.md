## Что и почему?
Начало нового проекта на C заключается в создании базовой структуры для вашего приложения. Программисты делают это, чтобы заложить фундамент, убедившись, что все последующее имеет аккуратное место для размещения.

## Как это сделать:
```C
#include <stdio.h>

int main() {
    printf("Привет, новый проект!\n");
    return 0;
}
```

Запустите, и вы должны увидеть:
```
Привет, новый проект!
```

## Погружение
В 1970-х годах родился язык C. Деннис Ритчи начал нечто великое в Bell Labs. Простота C делает его до сих пор популярным выбором для системного программного обеспечения, встраиваемых систем и высокопроизводительных приложений.

На начальном этапе выбирайте между процедурным или модульным стилями. Процедурный стиль прост и напоминает следование рецепту. Модульный позволяет организовать код на блоки – думайте о ингредиентах, сортированных по мисочкам. Оба подхода работают, но модульный лучше масштабируется для сложных проектов.

Под капотом, когда вы компилируете, ваша настройка превращается в исполняемый файл. Компилятор (например, GCC) воспринимает вашу функцию `main()` как точку входа. Но есть и другое: связывание библиотек, настройка makefile для более крупных проектов и, возможно, добавление некоторых директив препроцессора для вкуса.
