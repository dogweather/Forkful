## Что и почему?
Организация кода в функции заключается в разбиении кода на переиспользуемые блоки, которые выполняют конкретные задачи. Это делает код более понятным, удобным для отладки и поддержки.

## Как:
Давайте рассмотрим простой пример: скажем, вы хотите сложить два числа несколько раз.

Без функций:
```C
#include <stdio.h>

int main() {
    int sum1 = 5 + 3;
    printf("Sum1: %d\n", sum1);
    
    int sum2 = 2 + 8;
    printf("Sum2: %d\n", sum2);
    
    // Здесь могут быть дополнительные сложения...
    
    return 0;
}
```

С функциями:
```C
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int sum1 = add(5, 3);
    printf("Sum1: %d\n", sum1);
    
    int sum2 = add(2, 8);
    printf("Sum2: %d\n", sum2);
    
    // Используйте функцию add() для дополнительных сложений...
    
    return 0;
}
```

Вывод:
```
Sum1: 8
Sum2: 10
```

## Погружение в тему
До того как в C появились функции, программирование часто велось линейно, подобно рецепту. Но по мере роста программ, дублирование кода стало проблемой. Функции стали решением - они позволили нам выполнять один и тот же блок кода из разных частей программы без его повторного написания каждый раз. Это не только экономит место, но и время при внесении обновлений: измените функцию в одном месте, и каждая часть вашего кода, которая ее использует, получит обновление.

Альтернативы функциям могут включать вставной код, макросы или кодирование методом копирования и вставки, но это может привести к избыточному коду, подверженному ошибкам и трудному в обслуживании. В отличие от них, функции инкапсулируют функциональность, определяют четкие интерфейсы и могут уменьшить побочные эффекты при правильном использовании областей видимости.

Когда вы реализуете функции, обратите внимание на пару деталей: во-первых, старайтесь делать так, чтобы они выполняли только одно действие - это известно как принцип единственной ответственности. Во-вторых, названия важны - выбирайте описательные названия для функций и их параметров, чтобы ваш код был самодокументируемым.
