## Что и Почему?
Запуск нового проекта на C# означает настройку нового решения и файлов проекта, которые структурируют ваш код. Программисты начинают новые проекты для того, чтобы воплотить идеи в программное обеспечение, решить проблемы или исследовать технологии.

## Как:
Подкатим рукава и начнем работать с кодом. Предположим, у вас есть .NET 6 или более поздняя версия - это последнее на момент написания. Вы будете использовать .NET CLI для этого.

Создайте новое консольное приложение:
```C#
dotnet new console -o MyNewProject
```
Перейдите в директорию вашего проекта:
```C#
cd MyNewProject
```
Запустите ваш новый, шаблонный "Hello World":
```C#
dotnet run
```
Вы должны увидеть:
```
Hello, World!
```
Ваш новый проект начал свое движение!

## Подробнее
Раньше, вероятно, вы бы запустили Visual Studio и прошли бы через мастер. Но теперь - .NET CLI это ваш выбор. Это быстро и не предполагает многое о вашей среде разработки.

Альтернативы? Без сомнений. Visual Studio все еще там для графического интерфейса пользователя. Rider и Visual Studio Code также являются хорошими выборами. Но CLI? Это все о том, чтобы быть лаконичным и эффективным в написании скриптов.

Детали реализации? Ваш файл `.csproj` - ключ к королевству. Это XML, но не волнуйтесь - он в основном обслуживает себя сам. Здесь находится информация, необходимая вашему процессу сборки - целевая платформа, зависимости, ссылки на проекты, все хорошее.
