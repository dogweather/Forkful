## কি এবং কেন?
কোডকে ফাংশনের মধ্যে সাজানো মানে আপনার কোডকে পুনরায় ব্যবহারযোগ্য খণ্ডে ভাগ করা, প্রতিটি খণ্ড নির্দিষ্ট একটি কাজ করে। প্রোগ্রামাররা এটি করে থাকেন কারণ এতে কোড পড়া, ডিবাগ করা এবং পুনরায় ব্যবহার করা সহজ হয়। এটা লেগোগুলিকে বিনগুলিতে সাজানোর মতো - এটি আপনাকে প্রতিবার কিছু নির্মাণ করার জন্য একটি অব্যবস্থিত স্তুপ থেকে খুঁজে বের করার হ্যাপা থেকে রক্ষা করে।

## কিভাবে:
ধরুন আপনি একটি LED ঝলসাতে চান। ফাংশন ছাড়া, আপনার `loop` একটি অগোছালো জটিলতা। ফাংশন সহ, এটি পরিপাটি। এই হল কিভাবে:

```Arduino
const int LED_PIN = 13;

void setup() {
  pinMode(LED_PIN, OUTPUT);
}

void loop() {
  blinkLED(500); // ৫০০ মিলিসেকেন্ড অন্তর এলইডি ঝলসান
}

// একটি LED ঝলসানোর ফাংশন
void blinkLED(int delayTime) {
  digitalWrite(LED_PIN, HIGH);
  delay(delayTime);
  digitalWrite(LED_PIN, LOW);
  delay(delayTime);
}
```

নমুনা আউটপুট: আপনার LED আনন্দিতভাবে ঝলসাছে, এবং কোডের উদ্দেশ্য এক নজরে স্পষ্ট।

## বিস্তারিত আলোচনা
ফাংশন প্রবর্তনের আগে, প্রোগ্রামিং ছিল একটি রৈখিক রোড ট্রিপ; আপনি শুরু থেকে শেষ পর্যন্ত প্রতিটি খানাখন্দ দেখতে পেতেন। ফাংশনের পরে, এটি আরও বেশি মনে হয় ফ্লাইট হপিং-এর মতো - আপনি গুরুত্বপূর্ণ অংশগুলিতে সরাসরি চলে যান। ঐতিহাসিকভাবে, সাবরুটিন (প্রথমিক ফাংশন) প্রোগ্রামিং-এ এক বিপ্লব ছিল, যা কোডারদের নিজেদেরকে পুনরাবৃত্তি থেকে রক্ষা করতে সাহায্য করেছিল – এটি DRY নীতি, অর্থাৎ নিজেকে পুনরাবৃত্তি করো না। ফাংশনের বিকল্পগুলি অন্তর্ভুক্ত করতে পারে ম্যাক্রোস অথবা অবজেক্ট-ওরিয়েন্টেড প্রোগ্রামিং (OOP) এর জন্য ক্লাসের ব্যবহার। মূল কথা? আপনি যখন একটি ফাংশন সংজ্ঞায়িত করেন, আপনি কম্পাইলারকে একটি কাজ সম্পাদনের জন্য একটি নীলনকশা দিচ্ছেন। Arduino এর সাথে, আপনি প্রায়ই ভয়েড ফাংশন সংজ্ঞায়িত করে থাকেন যা একটি মাইক্রোকন্ট্রোলারের জন্য সাধারণ কমান্ড হোটে, কিন্তু ফাংশন মান ফেরত দিতে পারে, যা তাদেরকে আরও বহুমুখী করে তোলে।
