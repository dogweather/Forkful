## কি এবং কেন?

সি ভাষায় স্ট্যান্ডার্ড এররে লিখা মানে হলো এরর বার্তা এবং ডায়াগনস্টিক তথ্য মূল প্রোগ্রাম আউটপুট থেকে আলাদা একটি স্ট্রিমে প্রেরণ করা। প্রোগ্রামাররা এটি করেন এরর বার্তাগুলি স্ট্যান্ডার্ড আউটপুট থেকে বিচ্ছিন্ন করার জন্য, যা উভয়কে আলাদাভাবে পড়া এবং প্রক্রিয়া করা সহজ করে তোলে, বিশেষ করে ডিবাগিং অথবা প্রোগ্রামের এক্সিকিউশন লগ করার সময়।

## কিভাবে:

সি ভাষায়, `stderr` স্ট্রিমটি এরর বার্তা লেখার জন্য ব্যবহৃত হয়। স্ট্যান্ডার্ড আউটপুটে `printf` ব্যবহার করে লেখার মতো নয়, `stderr`-এ লিখতে `fprintf` অথবা `fputs` ব্যবহৃত হয়। এখানে এটি কিভাবে করবেন তা দেখানো হলো:

```c
#include <stdio.h>

int main() {
    fprintf(stderr, "এটি একটি এরর বার্তা।\n");

    fputs("এটি আরেকটি এরর বার্তা।\n", stderr);
    
    return 0;
}
```

স্যাম্পল আউটপুট (stderr-এ):
```
এটি একটি এরর বার্তা।
এটি আরেকটি এরর বার্তা।
```

উল্লেখ্য, যদিও কনসোলে আউটপুট `stdout`-এর মতো মনে হয়, টার্মিনালে রিডাইরেকশন ব্যবহৃত হলে পার্থক্যটা স্পষ্ট হয়ে ওঠে:

```sh
$ ./your_program > output.txt
```

এই কমান্ডটি কেবল স্ট্যান্ডার্ড আউটপুটকে `output.txt` ফাইলে রিডাইরেক্ট করে, অন্যদিকে এরর বার্তাগুলি এখনও স্ক্রিনে প্রদর্শিত হবে।

## গভীরে যাওয়া

`stdout` এবং `stderr`-এর মধ্যে পার্থক্য ইউনিক্স-ভিত্তিক সিস্টেমে সি এবং ইউনিক্সের প্রারম্ভিক দিনগুলি থেকে চলে আসছে। এই বিভাজন আরও দৃঢ় এরর হ্যান্ডলিং এবং লগিং সম্ভব করে তোলে, যা প্রোগ্রামারদের স্ট্যান্ডার্ড প্রোগ্রাম আউটপুট থেকে স্বাধীনভাবে এরর বার্তাগুলি রিডাইরেক্ট করার সুযোগ দেয়। যদিও `stderr` ডিফল্টভাবে আনবাফার্ড হয়ে থাকে যাতে এরর বার্তাগুলি অবিলম্বে আউটপুট হয়, যা ক্র্যাশ এবং অন্যান্য জরুরি ইস্যুগুলি ডিবাগ করার সময় সহায়ক হয়, `stdout` সাধারণত বাফার্ড হয়, অর্থাৎ এর আউটপুট বিলম্বিত হতে পারে যতক্ষণ না বাফার ফ্লাশ হয় (যেমন, প্রোগ্রাম সম্পূর্ণ হওয়া বা ম্যানুয়াল ফ্লাশিং)।
